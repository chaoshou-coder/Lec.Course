### Day 01 · 你的第一个 HTML 文档

> **痛点**:你看过无数网页,但从未想过网页的"源代码"长什么样。今天你将写出人生第一个 HTML 文档 —— 一个真正在浏览器里渲染出来的页面。
> **类比**:HTML 就像建筑的图纸。图纸本身不是房子,但工人按图纸建出房子。浏览器按 HTML 图纸"建"出网页。
> **解释**:HTML(HyperText Markup Language)是一种标记语言,用**标签**告诉浏览器"这段文字是标题"、"那张图片放在这里"。

---

#### 标签是什么 —— 网页的"词汇"

**逐行解剖**

HTML 的基本单位是**标签(tag)**,用尖括号包裹:

```html
<h1>欢迎来到我的网站</h1>
```

- `<h1>` = **开标签**(opening tag),表示"从这里开始,h1 级别的标题"
- `欢迎来到我的网站` = 内容(content)
- `</h1>` = **闭标签**(closing tag),斜杠 `/` 表示"到这里结束"

**为什么:**标签就像给内容"贴标签"。`<h1>` 告诉浏览器"这是最重要的标题",浏览器就用大号粗体渲染。

> **问自己:**
> 1. 如果忘记写闭标签 `</h1>`,会发生什么?
> 2. `<h1>` 和 `<h2>` 有什么区别?
> 3. 标签名区分大小写吗?

---

#### 四种标签形态

```html
<!-- 1. 开标签 + 内容 + 闭标签(最常见) -->
<p>这是一个段落</p>

<!-- 2. 自闭合标签(没有内容,自己关闭) -->
<br />
<img src="photo.jpg" />

<!-- 3. 带属性的标签 -->
<a href="https://example.com">点击这里</a>

<!-- 4. 空标签(有开无闭,HTML5 允许) -->
<meta charset="utf-8">
```

**逐行解剖**

- `<br />` = 换行符,不需要内容,所以自闭合。斜杠 `/` 在 HTML5 中可写可不写,但推荐写
- `<img src="photo.jpg" />` = 图片标签,`src` 是**属性**(attribute),告诉浏览器图片在哪里
- `<a href="...">` = 链接标签,`href` 属性指定跳转地址
- `<meta charset="utf-8">` = 元数据标签,声明文档用 UTF-8 编码(支持中文)

---

#### 属性 —— 标签的"参数"

**逐行解剖**

属性 = 键值对,写在开标签里,格式为 `属性名="属性值"`:

```html
<img src="images/photo.jpg" alt="我的头像" width="200">
```

- `src="images/photo.jpg"` = 图片路径(source)
- `alt="我的头像"` = 替代文本(图片加载失败时显示)
- `width="200"` = 宽度 200 像素

**常见错误**

> 1. **错误现象**:`src=photo.jpg`(没加引号)
>    **原因:**属性值必须用引号包裹,否则浏览器解析出错
> 2. **错误现象**:`<img src = "photo.jpg">`(等号两边有空格)
>    **原因:**虽然浏览器容错,但规范写法是紧贴等号

#### 文档基本结构 —— 每个 HTML 文件的"骨架"

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="utf-8">
    <title>我的第一个网页</title>
</head>
<body>
    <h1>欢迎来到我的网站</h1>
    <p>这是我的第一个 HTML 文档。</p>
</body>
</html>
```

> **ASCII 内存图**
> ```
> 文档根
> ├── <!DOCTYPE html>     ← "这是 HTML5 文档"
> └── <html lang="zh-CN">  ← 根元素,lang 声明语言
>      ├── <head>          ← 头部(浏览器不看,但必需)
>      │    ├── <meta charset="utf-8">  ← 编码声明
>      │    └── <title>我的第一个网页</title>  ← 标题栏显示
>      └── <body>          ← 身体(浏览器渲染的部分)
>           ├── <h1>欢迎来到我的网站</h1>
>           └── <p>这是我的第一个 HTML 文档。</p>
> ```

**逐行解剖**

- `<!DOCTYPE html>` = 文档类型声明,必须是第一行,告诉浏览器"按 HTML5 标准解析"
- `<html lang="zh-CN">` = 根元素,整个文档的容器。`lang="zh-CN"` 声明内容是中文(帮助搜索引擎和读屏软件)
- `<head>` = 头部,包含元数据(浏览器不直接渲染,但必需)
- `<meta charset="utf-8">` = 字符编码声明,必须写!否则中文可能乱码
- `<title>` = 浏览器标签页上显示的文字
- `<body>` = 身体,所有可见内容都放这里

**为什么:**这个骨架就像人的身体结构 —— head 是大脑(不直接可见但控制一切),body 是身体(你看到的部分),DOCTYPE 是出生证明(声明"我是谁")。

> **问自己:**
> 1. 如果忘记写 `<meta charset="utf-8">`,中文会怎样?
> 2. `<title>` 里的内容会显示在页面的哪个位置?
> 3. 为什么 `<head>` 里的内容不会显示在页面上?

#### 执行过程跟踪(关键)

```html
<!-- --- 执行过程 --- -->
<!-- 浏览器从上到下读取: -->

<!-- 第 1 行 <!DOCTYPE html>: -->
<!--   ① 识别:这是 HTML5 文档,按 HTML5 标准解析 -->

<!-- 第 2 行 <html lang="zh-CN">: -->
<!--   ① 创建根元素,语言 = 中文 -->

<!-- 第 3-6 行 <head> 内部: -->
<!--   ① <meta charset="utf-8">:设置编码为 UTF-8,支持中文 -->
<!--   ② <title>:把"我的第一个网页"设为标签页标题 -->

<!-- 第 7-10 行 <body> 内部: -->
<!--   ① <h1>:用一级标题样式渲染"欢迎来到我的网站" -->
<!--   ② <p>:用段落样式渲染"这是我的第一个 HTML 文档。" -->

<!-- 最终:浏览器标签页显示"我的第一个网页",页面内显示一个大标题 + 一个段落 -->
```

---

#### 常见错误

> 1. **错误现象**:中文显示为乱码(涓枃)
>    **原因:**忘记写 `<meta charset="utf-8">`,浏览器用错误编码解析中文
> 2. **错误现象**:页面空白
>    **原因:**内容写在了 `<head>` 里而不是 `<body>` 里
> 3. **错误现象**:标题没有变大
>    **原因:**用了 `<h7>`(不存在),HTML 只有 h1-h6
> 4. **错误现象**:标签名大写 `<H1>`
>    **原因:**HTML5 不区分大小写,但规范要求小写
