#### 为什么需要循环 —— 消灭重复代码

> **痛点**

同样语句复制粘贴 10 份,改需求要手动改 10 处,
一处漏改出 bug。

> **类比**

手写 100 份请帖 vs 打印机印 100 份,
循环 = 代码的打印机。

> **解释**

**DRY 原则** = Don't Repeat Yourself,
同一动作重复 N 次,用循环代劳。

**ASCII 对比**

```
复制粘贴:                    循环:
┌────────────────┐          ┌─────────────────────┐
│ 10 行同段代码  │          │ for _ in range(N):  │
│ 改 10 处       │          │     同一段代码      │
└────────────────┘          │     ← 只写一次     │
                            └─────────────────────┘
                              改 1 处,多处生效
```

```python
# 写法 A 复制粘贴 (反例)
print("打卡成功")
print("打卡成功")
print("打卡成功")
print("打卡成功")
print("打卡成功")
print("------------")

# 写法 B 循环 (正例)
for _ in range(5):
    print("打卡成功")

# 要点
# ① for _ in range(5): _ 占位变量,"不关心当前值"
# ② range(5) 生成 0,1,2,3,4 共 5 个数
# ③ 循环体缩进 4 格,for 共跑 5 次
```

**常见错误**

> 1. **错误现象**:`for i in range(3):print("x")` → IndentationError
>    **原因:**循环体必须缩进
> 2. **错误现象**:以为 `range(5)` 含 5
>    **原因:**实际不含,只到 4,左闭右开

> **问自己**:
> - 打印 200 次,写法 A/B 各改几行?
> - `range(5)` 改 `range(3)`,输出变几行?
> - 循环体忘缩进,运行时报什么?

#### while 循环 —— "当...时就继续"

> **痛点**

不知循环多少次,只知"满足条件就继续"。

> **类比**

跑步机 —— 心率低于目标值时继续,达标就停。

> **解释**

`while 条件:` 条件 **True** 执行循环体,
**False** 退出循环。

**ASCII 判断流程**

```
      ┌────────────┐
      │  判断条件  │
      └──────┬─────┘
         T   │     F
      ┌──────┘     └─────┐
      ▼                 ▼
┌────────────┐      ┌──────────┐
│ 执行循环体 │      │ 跳出循环 │
└────┬───────┘      └──────────┘
     └─────┐
           │
       (再判断)
```

```python
# 示例:不停询问密码,直到输入 123 才停
password = ""
while password != "123":
    password = input("请输入密码:")
print("密码正确,欢迎!")

# 逐行解剖
# ① password = "" 初值为空
# ② while password != "123": 条件 True → 进循环
# ③ password = input(...) 等待输入
# ④ 回到 while 再判断:输入 "123" → False → 退出
```

**常见错误**

> 1. **错误现象**:`while password != "123"` → SyntaxError
>    **原因:**忘写冒号
> 2. **错误现象**:`while p = "123":` → SyntaxError
>    **原因:**赋值 `=` 和比较 `==` 混淆

> **问自己**:
> - 初值改成 "123",循环体执行几次?
> - `while` 和 `if` 语法相似处?
> - 何时用 while,何时用 for?

#### while True + break 组合

> **解释**

`while True:` 永远为真 → 无限循环,
循环结束条件放在循环体内部用 break 控制。

```python
# 示例:无条件循环 + 手动退出
while True:
    word = input("输入 quit 退出:")
    if word == "quit":
        break           # 跳出循环
print("你退出了")

# 逐行解剖
# ① while True: 永远为真 → 无限循环
# ② word = input(...) 等待输入
# ③ 输入 "quit" → 触发 break → 退出循环
# ④ 输入其他 → 继续循环
```

#### while 三要素 —— 初始化 / 条件 / 更新

> **痛点**

while 易错 —— 忘初值会 NameError,
忘更新会死循环,条件写反也是死循环。

> **类比**

跑步机三要素 —— 开机(初始化) /
设目标心率(条件) / 逐渐提速(更新)。

```python
# 打印 1 到 5 (while 实现)
count = 1              # 要素 1:初始化
while count <= 5:      # 要素 2:条件
    print(count)
    count = count + 1  # 要素 3:更新
print("循环结束")
```

---

#### 死循环警示

> **痛点**

忘记更新变量或条件写错,程序会永远跑下去。

`Ctrl+C` 可强制中断运行中的 Python 程序。

> **BREAK IT:忘记更新 → 死循环**

```python
# ============ BREAK IT 演示 ============
# count = 1
# while count <= 5:
#     print(count)
#     # 这里少一行,补上能让循环终止
# ============ END BREAK IT ============

# 当前执行过程(漏更新)
# count=1 → 输出 1 → 没更新还是 1
# 第 2 轮 count <= 5 (仍为 1) → True → 又输出 1
# ...永远重复 = 死循环
```

**常见错误**

> 1. **忘记更新**:条件永远 True → 死循环
> 2. **初始化放循环内**:每轮重置 → 死循环
> 3. **条件写反**:`count > 0` 且递增 → 永远 True

> **问自己**:
> - 第 4 行改 `count = count + 2`,输出?
> - 第 1 行放循环内会怎样?
> - 如何改成打印 5 到 1 倒着数?

#### for + range —— 知道循环次数时首选

> **痛点**

while 必须手写三要素,简单计数太繁琐。

> **类比**

食堂窗口已知 5 个(for i in range(5)),
每个窗口打一次饭,打完就走。

> **解释**

`for 变量 in range(N):` 变量依次取
0,1,...,N-1,共 N 次。

```python
# 打印 0 到 4
for i in range(5):
    print(f"当前 i = {i}")
print("结束")

# 逐行解剖
# ① range(5) 生成序列 0,1,2,3,4
# ② i 第一次取值 0
# ③ print(f"当前 i = {i}") 输出 "当前 i = 0"
# ④ i 自动取值 1,依次到 i=4,循环结束
```

**常见错误**

> 1. **以为 range(5) 含 5**:实际 0-4,左闭右开
> 2. **在循环中改 i**:修改无效,被 range 覆盖

#### range 的三种形式

> **痛点**

需指定起点、终点、步长时,记混参数含义。

> **类比**

跑道 —— **range(n)** 起点跑 n 步;
**range(a,b)** a 跑到 b;
**range(a,b,step)** 按步长跑。

**ASCII 三种形式对比**

```
range(5)            → [0, 1, 2, 3, 4]
range(1, 6)         → [1, 2, 3, 4, 5]
range(0, 10, 2)     → [0, 2, 4, 6, 8]
range(5, 0, -1)     → [5, 4, 3, 2, 1]
```

```python
# 形式 1: range(stop) —— 0 到 stop-1
for i in range(5):
    print(i, end=" ")    # 0 1 2 3 4
print()

# 形式 2: range(start,stop) —— 含 start,不含 stop
for i in range(1, 6):
    print(i, end=" ")    # 1 2 3 4 5
print()

# 形式 3a: step 正数 —— 按步长跳
for i in range(0, 10, 2):
    print(i, end=" ")    # 0 2 4 6 8
print()

# 形式 3b: step 负数 —— 倒序
for i in range(5, 0, -1):
    print(i, end=" ")    # 5 4 3 2 1
print()
```

**要点**:
- `range(5)` —— 1 参数,从 0,不含 5
- `range(1,6)` —— 2 参数,从 1,不含 6
- `range(0,10,2)` —— 3 参数,每次加 2
- `range(5,0,-1)` —— step 负,倒序,不含 0

**常见错误**

> 1. **以为 stop 可含**:`range(1,10)` 最后是 9 不是 10
> 2. **倒序 stop 写错**:想 5-1 却写 range(5,-1,-1) 多了 0

> **问自己**:
> - `range(3,8)` 几次?哪些数?
> - `range(10,0,-3)` 输出?
> - 输出 5,4,3,2,1 参数?

#### 嵌套循环 —— 循环里再写循环

> **痛点**

矩形、乘法表等"行列"结构,单层循环搞不定。

> **类比**

日历 —— 外层管"周",内层管"日"。

> **解释**

外层循环一次,**内层循环完整跑一圈**,
两层组合产生**行 × 列**效果。

**ASCII 嵌套执行轨迹**

```
外层 i=0 ──→ 内层 j=0 j=1 j=2 j=3 (内层完整跑完)
外层 i=1 ──→ 内层 j=0 j=1 j=2 j=3 (内层完整跑完)
外层 i=2 ──→ 内层 j=0 j=1 j=2 j=3 (内层完整跑完)
```

```python
# 3 行 4 列的星号
for i in range(3):       # 外层:管行数
    for j in range(4):   # 内层:管列数
        print("*", end=" ")
    print()              # 每行结束换行

# 逐行解剖
# ① 外层第 1 轮 i=0:
#   - 内层 j=0,1,2,3 输出 "* * * *"
#   - print() 换行
# ② 外层第 2 轮 i=1:内层完整跑 → 第 2 行
# ③ 外层第 3 轮 i=2:内层完整跑 → 第 3 行
```

> **BREAK IT:忘末尾 print() → 星号全挤一行**

```python
# ============ BREAK IT 演示 ============
# for i in range(3):
#     for j in range(4):
#         print("*", end=" ")
#     # print()  # ← 注释掉,再运行
# ============ END BREAK IT ============
```

**常见错误**

> 1. **忘末尾 print()**:星号全挤在一行
> 2. **缩进搞混**:内层要比外层多缩 4 格

> **问自己**:
> - 外层和内层谁先跑完整一圈?
> - 改成 5 行 5 列怎么改?
> - 如何只打第一行星号?

#### break —— 立刻退出整个循环

> **痛点**

满足条件后希望立刻结束,不再执行多余迭代。

> **类比**

翻书包找钥匙 —— 找到就停,不翻其余夹层。

> **解释**

`break` 终止**离它最近的**循环,
跳到循环后继续执行。

```python
# 在字符串中找第一个元音字母
word = "apple"
found = False
for ch in word:
    if ch in "aeiou":
        print(f"找到元音:{ch}")
        found = True
        break           # 找到就退出,不再继续
if not found:
    print("没找到元音")

# 逐行解剖
# ① 第 1 轮 ch='a':'a' 在 "aeiou" → True
#   - 输出 "找到元音:a",found=True
#   - break 立刻退出循环
# ② 后续 'p','p','l','e' 不再检查
```

> **BREAK IT:嵌套循环中 break 只退一层**

```python
# ============ BREAK IT 演示 ============
# break 只退内层,不影响外层
for i in range(3):
    for j in range(3):
        if j == 1:
            break       # 只退内层,外层继续
        print(f"i={i},j={j}")
    print("---外层一轮结束")
# ============ END BREAK IT ============

# 执行过程
# i=0: 内层 j=0 输出 → j=1 break → 退出内层
# i=1: 内层重新取 j=0 输出 → j=1 break → 再退
# i=2: 同样行为,外层共正常跑 3 轮
```

**常见错误**

> 1. **以为 break 退所有循环**:只退一层,外层继续
> 2. **break 后同层代码永远不执行**

> **问自己**:
> - word="banana" 第几轮 break?
> - break 后多余迭代还执行吗?
> - break 和 continue 区别?

#### continue —— 跳过本次,继续下一轮

> **痛点**

想跳过某些元素,不终止整个循环。

> **类比**

点名时跳过请假的人,其他人照样点名。

> **解释**

`continue` 跳过本轮循环体剩余代码,
直接进入下一轮判断,循环本身不退出。

```python
# 只打印奇数,偶数全部跳过
for i in range(1, 11):
    if i % 2 == 0:   # 偶数
        continue     # 跳过本轮
    print(i)

# 逐行解剖
# ① i=1: 奇数 → 不 continue → 输出 1
# ② i=2: 偶数 → continue → 跳过 print
# ③ i=3: 奇数 → 输出 3
# ④ ...i=9 输出, i=10 跳过,结束
```

**break vs continue 口诀**

```
break = "辞职" → 离开公司,不再来
continue = "请假" → 只休一天,明天继续
```

```python
# break vs continue 对比
print("break:", end=" ")
for i in range(1, 8):
    if i == 4:
        break         # i == 4 直接退出
    print(i, end=" ")
print()

print("continue:", end=" ")
for i in range(1, 8):
    if i == 4:
        continue      # i == 4 只跳过一轮
    print(i, end=" ")
print()

# break: 1 2 3  (到 4 就停)
# continue: 1 2 3 5 6 7  (跳过 4 继续)
```

**常见错误**

> 1. **把 continue 当 break**:后续仍继续,只跳当前
> 2. **continue 后写了重要代码**:永远不执行

> **问自己**:
> - continue 后循环还继续吗?
> - 改成 break 后行为怎样变?
> - continue 能用 if/else 替代吗?

#### 累加器与累乘器 —— 循环中累积结果

> **痛点**

求和、求乘积,单行 print 做不到。

> **类比**

存钱罐 —— 每轮投一枚硬币到 total,
不断累积就是最后总额。

> **解释**

用**累加变量**保存中间结果,
**加法初值 0,乘法初值 1**。

口诀:**加 0 乘 1**

```python
# 累加器: 求 1+2+...+10
total = 0               # 累加初值 = 0
for i in range(1, 11):
    total = total + i   # 每轮加到 total
print(f"1到10的和={total}")

# 逐行解剖
# 初值 total = 0
# i=1: 0+1=1   i=2: 1+2=3   i=3: 3+3=6
# i=4: 6+4=10  ... i=10: 45+10=55
```

```python
# 累乘器: 求 1×2×3×4×5 (即 5 的阶乘)
product = 1                # 累乘初值 = 1
for i in range(1, 6):
    product = product * i  # 每轮乘到 product
print(f"5 的阶乘={product}")

# 逐行解剖
# 初值 product = 1
# i=1: 1*1=1  i=2: 1*2=2  i=3: 2*3=6
# i=4: 6*4=24  i=5: 24*5=120
```

> **BREAK IT:累乘初值写成 0**

```python
# ============ BREAK IT 演示 ============
# 故意写错: 累乘初值 = 0
product_wrong = 0
for i in range(1, 6):
    product_wrong = product_wrong * i
print(f"累乘初值=0,结果={product_wrong}")  # 必定是 0
# ============ END BREAK IT ============

# 执行过程
# i=1: 0*1=0  i=2: 0*2=0
# ... 永远乘不出非零值!
```

**常见错误**

> 1. **累乘初值写成 0**:0 乘任何数都为 0
> 2. **累加初值写成 1**:结果多 1

> **问自己**:
> - 结束后 total 存的是什么?
> - 初值写在循环内会怎样?
> - 为何累加初 0,累乘初 1?

#### 综合练习 —— 九九乘法表

> **目标**:用嵌套循环完成经典题目,
> 外层管行,内层管列,列数随行递增。

**ASCII 输出形状**

```
行1:  1*1=1
行2:  1*2=2  2*2=4
行3:  1*3=3  2*3=6  3*3=9
每行比上一行多一式子 → 下三角
```

```python
# 9×9 乘法表
for i in range(1, 10):
    for j in range(1, i + 1):
        print(f"{j}*{i}={i*j}", end="\t")
    print()                       # 每行结束换行

# 逐行解剖
# ① 外层 range(1,10) 共 9 行
# ② 内层 range(1,i+1) 第 i 行有 i 个式子
# ③ end="\t" 用制表符对齐,不换行
# ④ 内层结束 print() 换行
```

#### 今日小结

| 概念 | 解决的痛点 |
| --- | --- |
| while 条件循环 | 不知次数,True 就继续,False 退出 |
| for + range 计数循环 | 已知次数,比 while 简洁 |
| 嵌套循环 行列结构 | 矩形、表格式数据 |
| break 立刻退出 | 找到目标立刻停,不浪费时间 |
| continue 跳过本次 | 过滤特定元素,不终止整个循环 |
| 累加/累乘器 累积结果 | 求和求积,加 0 乘 1 |

> 易错点:while 缺步进变死循环;
> 累乘初值写成 0 结果恒 0;
