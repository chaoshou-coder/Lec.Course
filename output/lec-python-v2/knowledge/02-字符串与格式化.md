#### 字符串创建 —— 三种引号

> **痛点**

文字里也有引号怎么办?
比如 `悟空说:"俺老孙来也"`,
引号和引号打架,Python 不知道哪个是边界。

> **类比**

单/双引号就像左右手,完全等价,选一种即可。
当文字本身含引号,用另一种引号框住就行了。

> **解释**

用**单引号** `'...'`、**双引号** `"..."`、
**三引号** `'''...'''` 三种方式创建字符串。

**ASCII 内存图**

```
s = "悟空" → s ──→ 连续内存块:字符 '悟'(0) '空'(1)
```

**关键**:字符按顺序排,每人编号 0、1、2……

```python
# 示例 1:三种引号
s1 = '单引号'
s2 = "双引号"
s3 = '''三引号'''
print(s1, s2, s3)

# 逐行解剖
# ① 单引号(')与双引号(")完全等价
# ② 三引号支持跨行(多行字符串)
```

```python
# 示例 2:引号嵌套
line1 = '悟空说:"俺老孙来也"'
line2 = "她说:'你好'"
print(line1)
print(line2)

# 逐行解剖
# ① 外层用单引号,内部双引号是普通字符
# ② 外层用双引号,内部单引号是普通字符
```

**常见错误**

> 1. **错误现象**:`'悟空说:'你好''` → SyntaxError
>    **原因:**同类引号不能嵌套同类。修正:`'悟空说:"你好"'`
> 2. **错误现象**:文字中 `\` 被当成转义符
>    **原因:**`\n` `\t` 是转义字符。修正:`\\n` 表示字面 \n

> **问自己**:
> - 单引号双引号真的等价吗?
> - 文字里既有单引号又有双引号怎么办?
> - `\\` 输出是什么?

#### 索引 s[i] —— 给每个字符编门牌号

> **痛点**

"孙悟空"要取"悟"(第二个字),怎么做?

> **类比**

队伍同学编号 —— 正数 0 起,负数 -1 倒数。

> **解释**

`s[0]` = 第一个字符; `s[-1]` = 最后一个字符。

**ASCII 内存图**

```
s = "孙悟空"
正数索引:  0('孙')  1('悟')  2('空')
负数索引:    -3       -2       -1
```

```python
# 示例:正数索引 + 负数索引
name = "孙悟空"
print(name[0])     # 孙
print(name[1])     # 悟
print(name[2])     # 空
print(name[-1])    # 空
print(name[-2])    # 悟
print(name[-3])    # 孙

# 逐行解剖
# ① name[0] → 索引 0,即 '孙'
# ② name[-1] → 倒数第一,即 '空'
```

**常见错误**

> 1. **错误现象**:`name[3]` → IndexError(越界)
>    **原因:**只有 3 个字符,最大索引是 2
> 2. **错误现象**:以为第 N 个字符索引是 N
>    **原因:**第 N 个字符索引是 N - 1

> **问自己**:
> - "Python" 第 4 个字符索引怎么写?
> - `s[-1]` 和 `s[len(s)-1]` 相同吗?
> - 单字符时 `s[1]` 会怎样?

#### 切片 s[i:j] —— 一次截取一串

> **痛点**

从邮箱 `sunwu@163.com` 取用户名/域名,
一个一个字符取太麻烦。

> **类比**

切香肠 — 刀起 start,刀落 end,
**包含 start, 不包含 end**(左闭右开)。

> **解释**

`s[start:end]`,省略 start 默认 0,
省略 end 默认末尾。

**ASCII 内存图**

```
s = "0123456789"
s[2:6] 范围:2 3 4 5 (6 不包含)
结果: "2345"  (长度 6-2 = 4)

口诀:刀口落在 end,那块不拿。
```

```python
# 示例:完整切片 + 省略 start/end
s = "abcdefghij"
print(s[2:6])    # cdef
print(s[:3])     # abc
print(s[7:])     # hij
print(s[:])      # abcdefghij(复制整串)

# 逐行解剖
# ① s[2:6] → 索引 2、3、4、5 → "cdef"
# ② s[:3]  → 等价 s[0:3] → "abc"
# ③ s[7:]  → 从 7 到末尾 → "hij"
```

**常见错误**

> 1. **错误现象**:把 end 理解包含
>    **原因:**`s[1:4]` 取 1、2、3,**不含 4**
> 2. **错误现象**:起始 > 结束
>    **原因:**返回空串(不报错),不是倒序

> **问自己**:
> - `s[3:3]` 返回什么?
> - `s[:]` 和 `s` 是同一对象吗?
> - 怎么取前 3 个字符?

#### 步长 s[::k] —— 跳着取、倒着取

> **痛点**

如何每隔一个取一个,或把字符串倒过来?

> **类比**

每步跨 2 格(step=2);倒退着走(step=-1)。

> **解释**

`s[::2]` 跳取;`s[::-1]` 反转,
是反转的经典写法。

**ASCII 内存图**

```
s = "abcdefghijk" (索引 0~10)
s[::2]  → acegik (正步长跳取)
s[::-1] → kjihgfedcba (反转)
```

```python
# 示例:正步长 + 负步长
s = "abcdefghijk"
print(s[::2])      # acegik
print(s[::-1])     # kjihgfedcba

# 逐行解剖
# ① s[::2] → 0、2、4、6、8、10 → "acegik"
# ② s[::-1] → 从右往左逐个取 → 反转
```

**常见错误**

> 1. **错误现象**:`s[::0]` → ValueError
>    **原因:**step 不能为 0
> 2. **错误现象**:以为 s[1:5:-1] 能倒序取
>    **原因:**倒序时 start 必须 > end

> **问自己**:
> - `s[::-1]` 能判断回文吗?
> - `s[::-2]` 输出什么?
> - step = -1 时,start 默认是什么?

#### len() 求长度

> **痛点**

判断手机号是否 11 位,要先知道串长。

> **类比**

用刻度尺量一下。

> **解释**

`len(str)` 返回字符个数,一个中文也算 1。

```python
# 示例:len() 基本
s1 = "hello"
s2 = "悟空"
s3 = ""
print(len(s1))     # 5
print(len(s2))     # 2
print(len(s3))     # 0

# 逐行解剖
# ① len() 内置函数,不需 import
# ② 一个中文长度算 1
# ③ 空字符串长度 0
```

**常见错误**

> 1. **错误现象**:`"")[0]` → IndexError
>    **原因:**空串无索引
> 2. **错误现象**:`len(s)` 返回值当索引用
>    **原因:**合法索引范围是 0 ~ len(s) - 1

#### in 判断包含

> **痛点**

想知道文本是否含关键字"优惠",怎么做?

> **类比**

放大镜找关键词 — 找到 True,找不到 False。

> **解释**

`"X" in s` 返回 True/False,
`not in` 是其反义。

```python
# 示例:in / not in
text = "今天的优惠活动开始了"
print("优惠" in text)      # True
print("打折" in text)      # False
print("打折" not in text)  # True

# 逐行解剖
# ① "优惠" in text → 子串存在 → True
# ② "打折" not in text → 不存在 → True
```

**常见错误**

> 1. **错误现象**:`"" in "abc"` → True
>    **原因:**空串被认为是任何字符串的子串
> 2. **错误现象**:混淆 in 和 find
>    **原因:**`in` 返回 bool,`find` 返回位置

> **问自己**:
> - `"" in "abc"` 返回什么?为什么?
> - `in` 和 `find` 区别?
> - `in` 能否判断字符串是否不含敏感词?

#### find/rfind —— 查找子串位置

> **痛点**

知道子串存在,还要知道它在**哪里**。

> **类比**

find 正着翻书找第一次;rfind 倒着翻找最后一次。

> **解释**

`find(sub)` 返回第一次出现的索引,
`rfind(sub)` 最后一次,找不到返回 -1。

```python
# 示例:find / rfind
s = "hello hello hello"
print(s.find("hello"))    # 0
print(s.rfind("hello"))   # 12
print(s.find("world"))    # -1

# 逐行解剖
# ① find 从头开始找,找到第一个返回索引
# ② rfind 从末尾开始找,返回索引
# ③ 找不到返回 -1,不报错
```

**常见错误**

> 1. **错误现象**:`if s.find("X"):` 判断出错
>    **原因:**找不到时 -1 是真值,判断会反过来
> 2. **错误现象**:和 `index` 搞混
>    **原因:**`index` 找不到会报错,`find` 返回 -1

> **问自己**:
> - 如何判断字符串**不包含**某子串?
> - `find("X")` 和 `index("X")` 在找不到时有什么差别?
> - rfind 的应用场景是什么?

#### 字符串四件套 —— replace / strip / join / split

> **痛点**

敏感词替换、清理头尾空格、拼合/切开字符串。

> **类比**

replace = 换词,strip = 削皮,
join = 粘胶,split = 切开。

> **解释**

最常用的字符串处理工具,
配合使用能解决 80% 文字场景。

**ASCII 内存图**

```
text = "xxx 是个好网站,xxx 很好用"
fresh = text.replace("xxx", "***")

text  ──→ │xxx 是个好网站│(旧串,未变)
fresh ──→ │*** 是个好网站│(新串,replace 返回)

口诀:字符串不可变,replace 必须接返回值。
```

```python
# 示例:replace + strip + split + join
text = "xxx 是个好网站,xxx 很好用"
fresh = text.replace("xxx", "***")
print(text)      # 原串不变
print(fresh)     # 替换后的新串

s = "   hello   "
print(s.strip())    # hello (去头尾空白)

words = "a b c d"
print(words.split(" "))     # ['a', 'b', 'c', 'd']

joined = "、".join(["悟空", "八戒"])
print(joined)               # 悟空、八戒
```

**逐行解剖**

① `replace(old, new)` 把所有 old 换成 new,原串不变
② `strip()` 去头尾空白,中间不动
③ `split(sep)` 按 sep 切,返回列表
④ `sep.join(list)` 用 sep 拼合

**常见错误**

> 1. **错误现象**:不接 replace 返回值,以为原串被改
>    **原因:**字符串不可变,replace 返回新串
> 2. **错误现象**:strip 想去中间空格,去不掉
>    **原因:**strip 只能去头尾

> **问自己**:
> - `replace` 修改了原串吗?
> - 去中间空格用 strip 还是 replace?
> - `"->".join(["a","b","c"])` 输出什么?

#### 字符串不可变 —— 不能通过索引修改

> **痛点**

想把 "hello" 第 0 个字符改成 "H",
为什么报错?

> **类比**

字符串像**刻在石头上的字**,
想改只能重刻一块。

> **解释**

字符串是**不可变**(immutable)类型,
创建后不能通过索引修改。

**ASCII 内存图**

```
s = "hello" → s ──→ │ h  e  l  l  o │ (不可修改)
s[0] = "H" → TypeError
改法:s = "H" + s[1:] → 新串 │ H  e  l  l  o │
```

```python
# 示例:修改字符串的正确做法
s = "hello"
# s[0] = "H"  # 取消注释会报错 TypeError!
s = "H" + s[1:]
print(s)       # Hello

# 逐行解剖
# ① s[0] = "H" → TypeError,不支持索引赋值
# ② 通过切片+拼接构造新串
```

**常见错误**

> 1. **错误现象**:把字符串当成可变
>    **原因:**Python 字符串是 immutable
> 2. **错误现象**:以为 replace 修改了原串
>    **原因:**replace 也是返回新串

> **问自己**:
> - 为什么字符串要被设计成不可变?
> - 如果字符串可变, replace 还需返回新串吗?
> - `replace("x","y")` 不加赋值,原串会变吗?

#### f-string 格式化输出 —— 把变量嵌入字符串

> **痛点**

拼接 "我的名字是 X,今年 Y 岁" 这种句式,
手写 `+` 容易漏空格、忘转类型。

> **类比**

像**占位符** —— "姓名:____ 年龄:____",
运行时自动填充。

> **解释**

在字符串前加 **f**,花括号 **{}** 内放变量或表达式。

**ASCII 内存图**

```
name = "悟空"; age = 18
print(f"我是{name},今年{age}岁")
                 ↓       ↓
              替换为"悟空"  替换为18
结果:"我是悟空,今年18岁"
```

```python
# 示例:f-string 基本 + 表达式
name = "悟空"
age = 18
print(f"我是{name},今年{age}岁")

# 花括号内可写表达式
a = 10; b = 3
print(f"{a} + {b} = {a + b}")
print(f"{a} × {b} = {a * b}")

# 逐行解剖
# ① 前缀 f 表示这是 f-string
# ② {name} 会被替换为变量值
# ③ 花括号内可写表达式,如 {a + b}
```

**常见错误**

> 1. **错误现象**:漏前缀 f
>    **原因:**花括号原样输出,不替换
> 2. **错误现象**:花括号不配对
>    **原因:**要输出字面 `{}`,写 `{{}}`

> **问自己**:
> - f-string 和 `+` 拼接比,优点是什么?
> - 若要输出字面 "{}",怎么写?
> - 花括号内能放函数调用吗?

#### f-string 进阶 —— 对齐 / 小数

> **痛点**

默认 f-string 对不齐,小数位数不确定时怎么格式化?

> **类比**

像 Excel 单元格格式:居左/右、保留 N 位小数。

> **解释**

花括号内写 `变量:格式`,
`{pi:.2f}` 保留两位,`{n:>10}` 右对齐宽度 10。

```python
# 示例:保留 N 位小数 + 对齐
pi = 3.14159265
print(f"圆周率约 {pi:.2f}")     # 3.14
print(f"圆周率约 {pi:.4f}")     # 3.1416

# 对齐
print(f"{'书名':>8} {'价格':>8}")
print(f"{'Python 入门':>8} {'59.8':>8}")

# 逐行解剖
# ① :.N f → 保留 N 位小数(四舍五入)
# ② :>N   → 右对齐,占 N 字符宽
```

**常见错误**

> 1. **错误现象**:`{:2f}` 少写点
>    **原因:**被当宽度 2,应是 `{:.2f}`
> 2. **错误现象**:对齐方向写反
>    **原因:**`<` 左对齐,`>` 右对齐,`^` 居中

> **问自己**:
> - `{:.2f}` 和 `{.2f}` 哪个对?
> - `{n:>10}` `<` `>` `^` 分别什么对齐?
> - `a = 3.14159; print(f"{a:.2f}")` 输出什么?

| len | 字符个数 | 中文也算 1 |
| in | 判断子串存在 | "X" in s → True/False |
| find/rfind | 找子串位置 | 找不到返回 -1 |
| 四件套 | 替换/去空白/拼合/切开 | replace / strip / join / split |
| 不可变 | 不能通过索引修改 | 必须构造新串 |
| f-string | 嵌入变量 | f"{x}" |
| f-string 格式 | 对齐与小数 | :.N f / :>N |
| 综合 | 脱敏/回文/表格/URL | slicing + join + find + f-string |
