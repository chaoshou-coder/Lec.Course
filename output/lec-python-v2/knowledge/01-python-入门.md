#### print() —— 让电脑开口说话

> **痛点**

程序跑起来黑漆漆一片,完全不知道它在干什么。
需要一种方式让电脑"汇报"信息。

> **类比**

`print()` 就像一个对话框,括号里写啥,
屏幕上就显示啥。

> **解释**

`print()` 是 Python 最常用的输出函数。
把内容放进括号,电脑就会把它**打印**到屏幕上。

**ASCII 内存图 —— print 的执行过程**

```
print("Hello World")

执行过程:
  ① 从括号内取出字符串 "Hello World"
  ② 把字符串发送到标准输出(屏幕)
  ③ 默认在末尾自动加一个换行 \n

屏幕:
  Hello World
  ← 光标移到下一行
```

**关键**:`print()` 是函数调用,括号不能为空,
至少要打印一个值(也可以空括号换行)。

```python
# 示例 1:print() 的四种用法
# 用法 1:打印一个字符串
print("Hello World")        # Hello World

# 用法 2:输出中文 —— Python 3 原生支持
print("你好,我是 Python")    # 你好,我是 Python

# 用法 3:一次打印多个值 —— 逗号分隔,Python 自动加空格
print("刘备", "关羽", "张飞")  # 刘备 关羽 张飞

# 用法 4:字符串拼接 —— + 号粘在一起,没有空格
print("刘" + "关" + "张")    # 刘关张
```

**逐行解剖**

① `print("Hello World")` → 打印一个字符串
② `print("你好,我是 Python")` → 中文直接写
③ `print("刘备", "关羽", "张飞")` → 多值用逗号分隔,自动加空格
④ `print("刘" + "关" + "张")` → `+` 拼接,没有空格

**要点**:逗号分隔会自动加空格,`+` 拼接不会,
根据需求选择合适的方式。

```python
# 示例 2:print() 空括号 —— 打印一个空行
print("第一行")
print()          # 打印一个空行
print("第三行")

# 逐行解剖
# ① print("第一行") → 输出 "第一行" 并换行
# ② print()         → 什么都不打印,只输出一个换行
# ③ print("第三行") → 输出 "第三行"
```

**常见错误**

> 1. **错误现象**:`print("Hello'` 报错 SyntaxError
>    **原因:**引号没有配对,开头是双引号,结尾是单引号。修正:`print("Hello")`
> 2. **错误现象**:`print "Hello"` 报错 SyntaxError
>    **原因:**Python 3 的 print 是函数,必须加括号。修正:`print("Hello")`

> **问自己**:
> - `print("A", "B")` 和 `print("A" + "B")` 输出有什么不同?
> - `print()` 空括号会输出什么?
> - 引号用单引号可以吗?

#### 转义字符与 end —— 控制输出的格式

> **痛点**

想在字符串中间换行、想把两段输出接在同一行,
直接写发现做不到。

> **类比**

转义字符就像"快捷键":
`\n` 是回车换行,`\t` 是 Tab 对齐,
`end=""` 可以取消 print 默认的末尾换行。

> **解释**

反斜杠 `\` 是转义字符前缀,
`\n` 表示换行,`\t` 表示制表符。
`end` 参数控制 print 末尾加什么,默认是 `\n`。

**ASCII 内存图 —— \n 的作用**

```
s = "第一行\n第二行"

内存中:
  字符: '第' '一' '行' ↵ '第' '二' '行'
                      ↑
                    \n 换行符(一个字符)

打印到屏幕:
  第一行
  第二行      ← \n 把一行拆成两行
```

**关键**:`\n` 是**一个字符**,不是两个,
它代表"换行"这个操作。

```python
# 示例 1:\n 换行 + \t 对齐
print("第一行\n第二行")
print("姓名\t年龄\t城市")
print("张三\t18\t北京")

# 逐行解剖
# ① print("第一行\n第二行") → \n 触发换行,输出两行
# ② print("姓名\t年龄\t城市") → \t 是 Tab,用于对齐
# ③ print("张三\t18\t北京")  → 表格式输出
```

```python
# 示例 2:end="" 取消默认换行
print("不换行", end="")
print("接续这一行")
print("---")
print("默认换行")
print("换到新的一行")

# 逐行解剖
# ① print("不换行", end="") → 末尾不加 \n
# ② print("接续这一行")    → 直接接在上一个输出后面
# ③ print("默认换行")      → 末尾默认加 \n,换到新行
```

**常见错误**

> 1. **错误现象**:`print("C:\new\test")` 输出错乱
>    **原因:**`\n` 被当成换行符。修正:`print("C:\\new\\test")` 或 `print(r"C:\new\test")`
> 2. **错误现象**:`end` 拼错成 `endd`
>    **原因:**关键字拼写错误。修正:`end=""`

> **问自己**:
> - `\n` 和 `\\n` 有什么区别?
> - `end=""` 的作用是什么?
> - 想输出字面反斜杠 `\`,应该怎么写?

#### input() —— 让电脑听你说话

> **痛点**

程序只能"自言自语",无法接收用户的输入,
只能显示固定内容。

> **类比**

`input()` 就像一个话筒 —— 你把说的话传给了程序,
程序可以根据你的输入做出不同的反应。

> **解释**

`input()` 让程序暂停,在屏幕上显示提示语,
等你从键盘输入内容并按回车。
输入的内容**永远**以字符串形式返回。

**ASCII 内存图 —— input 的执行过程**

```
name = input("请输入你的姓名:")

执行过程:
  ① 屏幕显示 "请输入你的姓名:" 并等待
  ② 用户输入 "小明" 并按回车
  ③ input() 返回字符串 "小明"
  ④ name = "小明"

内存:
  name ──→ "小明" (字符串)
```

**关键**:`input()` 返回的**永远是字符串**,
即使你输入数字 `18`,得到的也是 `"18"`。

```python
# 示例 1:input() 基本用法
name = input("请输入你的姓名:")
print("你好," + name)

# 逐行解剖
# ① input("请输入你的姓名:") → 显示提示,等待输入
# ② 用户输入 "小明" → input() 返回 "小明"
# ③ name = "小明" → 赋值给变量
# ④ print("你好," + name) → 拼接后输出 "你好,小明"
```

```python
# 示例 2:input() 返回的是字符串
age = input("请输入年龄:")
print(type(age))       # <class 'str'>
# print(age + 5)       # 报错!字符串 + 整数不允许

# 正确做法:用 int() 转换成整数再算
age_num = int(age)     # 字符串 "18" → 整数 18
print(age_num + 5)     # 可以正常算数了

# 逐行解剖
# ① age = input(...) → age 是字符串 "18"
# ② type(age) → 确认类型是 str
# ③ age_num = int(age) → int() 把字符串转成整数
# ④ age_num + 5 → 整数加法,结果 23
```

**常见错误**

> 1. **错误现象**:`print(age + 5)` 报 TypeError
>    **原因:**`age` 是字符串,不能直接和整数相加。修正:`print(int(age) + 5)`
> 2. **错误现象**:`input` 后忘写括号
>    **原因:**`input` 是函数,必须调用。修正:`input("提示")`

> **问自己**:
> - `input()` 返回的是什么类型?
> - 想让用户输入数字并做计算,需要怎么做?
> - `input()` 的括号里的提示语有什么作用?

#### 变量与赋值 —— 给数据起个名字

> **痛点**

数据用了一次就丢了,想重复使用某个值,
只能每次重新写,麻烦且容易出错。

> **类比**

**变量**就是贴了标签的盒子。
你把数据装进盒子,以后用名字就能找到它,
不需要记住每次的具体值。

> **解释**

`=` 不是数学里的"等于",而是**赋值**:
把右边的值放进左边的盒子。
`==` 才是比较。

**ASCII 内存图 —— 变量的存储方式**

```
name = "小明"
age = 18

内存:
  name ──→ "小明"     ← 变量名指向数据
  age  ──→ 18

要点:
  - 变量名 → 标签(贴在盒子上)
  - 变量值 → 盒子里的东西
  - = 赋值 → 把数据放进盒子
```

**关键**:变量可以被多次赋值,新值**覆盖**旧值。

```python
# 示例 1:创建变量
name = "小明"       # 字符串
age = 18            # 整数
height = 1.75       # 浮点数
is_student = True   # 布尔值

print(name, age, height, is_student)

# 逐行解剖
# ① name = "小明" → 创建一个字符串变量
# ② age = 18 → 创建一个整数变量
# ③ height = 1.75 → 创建一个浮点数变量
# ④ is_student = True → 创建一个布尔变量
```

```python
# 示例 2:变量可以被多次赋值
x = 10
print(x)       # 10
x = 20         # 用新值 20 覆盖旧值 10
print(x)       # 20

# 逐行解剖
# ① x = 10 → x 指向 10
# ② print(x) → 输出 10
# ③ x = 20 → x 改为指向 20(10 被丢弃)
# ④ print(x) → 输出 20
```

```python
# 示例 3:把一个变量的值赋给另一个变量
a = 10
b = a           # b 复制了 a 的值,此时 b 也是 10
a = 99          # 修改 a 不影响 b
print(b)        # 仍然是 10

# 逐行解剖
# ① a = 10 → a 指向 10
# ② b = a → b 复制 a 的值,b 也指向 10
# ③ a = 99 → a 改为指向 99,b 不受影响
# ④ print(b) → 输出 10
```

**常见错误**

> 1. **错误现象**:`print(price)` 报 NameError
>    **原因:**变量使用前必须先赋值。修正:先 `price = 100`
> 2. **错误现象**:`18 = age` 报 SyntaxError
>    **原因:**赋值左边必须是变量名。修正:`age = 18`

> **问自己**:
> - `=` 和 `==` 有什么区别?
> - 变量可以赋值多次吗?赋值后旧值去哪了?
> - `b = a` 后修改 a,b 会变吗?

#### 变量命名规则 —— 起一个"合法"的名字

> **痛点**

起变量名时随心所欲,结果 Python 报错,
不知道哪里不合法。

> **类比**

变量名就像一个人的名字:
只能包含字母、数字、下划线,不能以数字开头,
不能用保留关键字(if/for/print...)。

> **解释**

命名三件套:
1. 只能由**字母、数字、下划线**组成
2. **不能以数字开头**
3. 不能是**保留关键字**

**ASCII 合法 vs 非法对照**

```
合法:                 非法:
  user_name     ✓       2name       ✗ (数字开头)
  _age          ✓       my-name     ✗ (含横杠)
  name2         ✓       if          ✗ (保留字)
  userName      ✓       my name     ✗ (含空格)
```

**约定写法**:`snake_case`(小写+下划线),
例如 `user_name`。

```python
# 示例:合法 vs 非法变量名
user_name = "张三"    # ✓
_age = 18             # ✓
name2 = "李四"        # ✓
# 2name = "张三"      # ✗ 数字开头,报错
# my-name = "张三"    # ✗ 横杠不行,报错
# if = 18             # ✗ 保留字,报错

# 逐行解剖
# ① user_name → 字母+下划线,合法
# ② _age → 下划线开头,合法(但通常有特殊含义)
# ③ name2 → 数字不在开头,合法
# ④ 2name → 数字开头,非法
```

**常见错误**

> 1. **错误现象**:`my-name = 1` 报 SyntaxError
>    **原因:**横杠 `-` 不是合法字符。修正:`my_name = 1`
> 2. **错误现象**:`print = 1` 之后再 `print("hi")` 报错
>    **原因:**`print` 是内置函数名,赋值后会覆盖。修正:换个名字

> **问自己**:
> - 变量名 `2name` 为什么不合法?
> - 变量名区分大小写吗?
> - 可以用 `print` 做变量名吗?会有什么后果?

#### 注释 —— 给代码加说明

> **痛点**

代码写了一周后回来看,完全看不懂自己当初
想干什么;同事看你的代码也是一头雾水。

> **类比**

注释就像书页旁边的批注:
给读者解释"这段代码在干什么"、
"为什么这样写",不影响正文内容。

> **解释**

Python 中有两种注释:
- **单行注释**:以 `#` 开头,到行末结束
- **多行注释**(文档字符串):用三引号 `'''` 或 `"""`

**ASCII 内存图 —— 注释被 Python 忽略**

```python
# 这是一行注释
print("Hello")  # 这也是注释

执行时:
  ① # 这是一行注释 → 跳过,不执行
  ② print("Hello") → 执行
  ③ # 这也是注释  → 跳过,不执行
```

**关键**:注释是给**人看的**,Python 完全忽略。

```python
# 示例 1:单行注释
# 下面这行代码打印欢迎语
print("欢迎来到 Python 世界")

# 示例 2:行内注释
age = 18    # 定义变量 age,初始值为 18
print(age)  # 打印 age 的值

# 示例 3:多行注释(文档字符串)
"""
这是一个多行注释。
可以用来写较长的说明,
比如文件头、函数说明等。
"""
print("Hello")
```

**逐行解剖**

① `# 内容` → 单行注释,`#` 后面的内容不执行
② `代码  # 注释` → 行内注释,前面代码正常执行
③ `"""..."""` → 多行注释,三引号包裹

**要点**:
- 注释不要太啰嗦,解释"为什么"比"是什么"更有价值
- 不要写 `age = 18  # 把 18 赋值给 age` 这种废话注释

**常见错误**

> 1. **错误现象**:注释嵌套三引号报错
>    **原因:**`"""多行注释"""里再写"""会提前结束`。修正:分开写或换引号类型
> 2. **错误现象**:# 被嵌在三引号里
>    **原因:**三引号里 `#` 是普通字符,不是注释。修正:确保 # 在引号外面

> **问自己**:
> - 注释会影响程序的执行速度吗?
> - 注释应该解释"是什么"还是"为什么"?
> - 三引号注释和单行注释有什么区别?

#### 综合练习 —— 对话程序

现在把**print/input/变量**串成一个完整程序。

**需求清单**:
1. 用 `input()` 询问用户的姓名和年龄
2. 把输入存到变量中
3. 用 f-string 输出一句欢迎语(至少用一次 f-string)
4. 打印出的句子包含用户输入的内容

```python
# 示例:对话程序
# --- 执行过程 ---

# 询问姓名
name = input("请输入你的姓名:")
# 询问年龄(注意 age 是字符串)
age = input("请输入你的年龄:")

# 拼接字符串打招呼
print("你好," + name + "!你今年" + age + "岁了")

# 逐行解剖
# ① name = input(...) → 接收姓名
# ② age = input(...)  → 接收年龄
# ③ print("你好," + name + ...) → 拼接输出
```

**逐行解剖**

① `name = input(...)` 接收姓名并存入变量
② `age = input(...)` 接收年龄并存入变量
③ `print(... + ...)` 拼接多个字符串输出

**要点**:这就是"输入 → 处理 → 输出"的
最小完整模型,以后的程序都是这个模式的扩展。

> - 如果要计算"10 年后你多少岁",需要怎么做?
> - `print` 里可以用逗号代替 `+` 吗?效果一样吗?
