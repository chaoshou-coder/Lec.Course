#### 列表创建与索引切片 —— 把一排数据粘在一起

> **痛点**

之前用独立变量 `name1`, `name2`, `name3` 存多个数据,
变量散落在各处,无法批量操作。

> **类比**

**列表** = 一排带编号的储物柜,编号从 0 开始。
你只要说"第 3 号柜子",就能直接拿到里面的东西。

> **解释**

列表用**方括号** `[]` 表示,
元素之间用**逗号**分隔。
列表和字符串很像:**有序、支持索引和切片**。
关键区别:字符串不可变,**列表可以修改**。

**ASCII 内存图 —— 列表的存储结构**

```
fruits = ["苹果", "香蕉", "橙子", "葡萄", "西瓜"]

内存中:
  变量 fruits ──→  ┌─────┬─────┬─────┬─────┬─────┐
                   │ 苹果 │ 香蕉 │ 橙子 │ 葡萄 │ 西瓜 │
                   └─────┴─────┴─────┴─────┴─────┘
  索引(正):          0     1     2     3     4
  索引(负):         -5    -4    -3    -2    -1
```

**规则**:索引从 0 开始,负数从 -1 倒着数。

```python
# 示例 1:创建列表,用索引访问元素
fruits = ["苹果", "香蕉", "橙子", "葡萄", "西瓜"]
print(fruits[0])     # 苹果
print(fruits[2])     # 橙子
print(fruits[-1])    # 西瓜
print(fruits[-2])    # 葡萄

# 逐行解剖
# ① fruits[0] 从列表开头偏移 0 个位置 → "苹果"
# ② fruits[-1] 负数索引 = 长度 + (-1) = 4 → "西瓜"
```

```python
# 示例 2:切片 —— 取一段连续元素
fruits = ["苹果", "香蕉", "橙子", "葡萄", "西瓜"]
print(fruits[1:4])    # ['香蕉', '橙子', '葡萄']
print(fruits[:3])     # ['苹果', '香蕉', '橙子']
print(fruits[3:])     # ['葡萄', '西瓜']
print(fruits[::-1])   # 逆序

# 逐行解剖
# ① fruits[1:4] 起点索引 1(含),终点索引 4(不含)
# ② fruits[::-1] 步长 -1 = 从后往前取 → 逆序
```

**常见错误**

> 1. **索引越界**:`fruits[5]` → IndexError
>    **原因:**只有 5 个元素,最大索引是 4
> 2. **切片边界搞反**:以为 `fruits[1:4]` 包含索引 4
>    **原因:**含头不含尾,索引 4 不包含

> - `fruits[1:4]` 取出几个元素?
> - `fruits[::-1]` 和 `fruits` 是同一个列表吗?

#### 列表可变性 —— 直接改,不报错

> **痛点**

字符串不能修改单个字符:`s = "abc"; s[0] = "X"` 报错。
但现实中经常需要"更新第 3 个学生的成绩"这种操作。

> **类比**

**字符串** = 刻在石头上的字,刻了就改不了。
**列表** = 写在白板上的清单,随时可以擦掉重写。

> **解释**

列表是**可变对象**(mutable),
可以通过索引直接赋值修改。
这是列表和字符串的**核心区别**。

```python
# 示例 1:列表可变 —— 直接通过索引修改
fruits = ["苹果", "香蕉", "橘子"]
fruits[0] = "榴莲"
print(fruits)    # ['榴莲', '香蕉', '橘子']

# 逐行解剖
# ① 找到索引 0 的位置
# ② 把该位置的引用从 "苹果" 改为 "榴莲"
# ③ 列表长度不变,只是内容变了
```

```python
# 示例 2:对比 —— 字符串不可变
s = "abc"
# s[0] = "X"  # 取消注释会报错!
# TypeError: 'str' object does not support item assignment
s = "X" + s[1:]    # 重建新字符串
print(s)           # Xbc
```

> **BREAK IT:列表的共享引用陷阱**

```python
# ============ BREAK IT 演示 ============
# a = [1, 2, 3]
# b = a           # b 和 a 指向同一个列表!
# b[0] = 99
# print(a)        # [99, 2, 3] ← a 也被改了!
# ============ END BREAK IT ============

# 原因:b = a 是引用赋值,不是复制
# 修正:用 b = a[:] 或 b = list(a) 复制一份
```

**常见错误**

> 1. **修改列表以为会创建新列表**:
>    `fruits[0] = "X"` 是原地修改,列表地址不变
> 2. **对字符串用索引赋值**:
>    `s[0] = "X"` → TypeError

> **问自己**:
> - 列表和字符串都支持索引访问,为什么一个能改一个不能?
> - 如果 `a = [1,2]; b = a; b[0] = 9`,`a[0]` 是多少?
> - 字符串真的完全不能改吗?有什么间接办法?

#### 列表 CRUD(增删)—— append / insert / pop / remove

> **痛点**

创建列表后,经常需要动态添加或删除元素。
比如购物车:用户每买一件商品,就往列表里加一个。

> **类比**

列表是一个**队伍**,
`append` = 排在队伍末尾,
`insert` = 插队(指定位置),
`pop` = 从队伍里移除一个人。

> **解释**

CRUD = Create(增) / Read(查) / Update(改) / Delete(删)。
本节先讲**增**和**删**,改就是索引赋值(上节已学)。

**ASCII 内存图 —— append 和 pop 的过程**

```
fruits = ["苹果", "香蕉"]

append("橘子") 后:
  ┌─────┬─────┬─────┐
  │ 苹果 │ 香蕉 │ 橘子 │  ← 末尾追加
  └─────┴─────┴─────┘

pop() 后:
  ┌─────┬─────┐
  │ 苹果 │ 香蕉 │  ← 末尾被移除
  └─────┴─────┘
```

```python
# 示例 1:增 —— append / insert / extend
fruits = ["苹果", "香蕉"]
fruits.append("橘子")
print(fruits)    # ['苹果', '香蕉', '橘子']

fruits.insert(1, "榴莲")
print(fruits)    # ['苹果', '榴莲', '香蕉', '橘子']

fruits.extend(["葡萄"])
print(fruits)    # ['苹果', '榴莲', '香蕉', '橘子', '葡萄']

# 逐行解剖
# ① append("橘子") 末尾追加,长度 +1
# ② insert(1, "榴莲") 在索引 1 处插入,后续元素右移
# ③ extend(["葡萄"]) 合并另一个列表
```

```python
# 示例 2:删 —— pop / remove / clear
fruits = ["苹果", "香蕉", "橘子", "葡萄"]
last = fruits.pop()
print(last)      # 葡萄
print(fruits)    # ['苹果', '香蕉', '橘子']

first = fruits.pop(0)
print(first)     # 苹果
print(fruits)    # ['香蕉', '橘子']

fruits.remove("香蕉")
print(fruits)    # ['橘子']

# 逐行解剖
# ① pop() 删末尾并返回被删元素
# ② pop(0) 删索引 0,后续元素左移
# ③ remove("香蕉") 按值删第一个匹配项,找不到报错
```

**常见错误**

> 1. **pop 取值后以为原列表不变**:
>    `pop()` 会**修改原列表**,不是只取值
> 2. **remove 不存在的值**:
>    `remove("不存在的值")` → ValueError
> 3. **insert 索引超出范围**:
>    不会报错,超出就等价于 append

> **问自己**:
> - `pop()` 和 `pop(0)` 哪个更快?为什么?
> - 如果要删所有值为 "X" 的元素,用 remove 够吗?
> - `append` 和 `extend` 有什么区别?

#### 列表排序与查询 —— sort / index / count / in

> **痛点**

拿到一堆数据后,经常需要:
排序(从高到低看排名)、查找(某个值在哪)、
计数(某个值出现几次)、判断(某个值在不在)。

> **类比**

列表是一副扑克牌:
`sort` = 按大小排一下,
`index` = 找到红桃 A 在第几张,
`count` = 数一下有几个 A,
`in` = 看看这副牌里有没有大王。

```python
# 示例 1:排序 —— sort (原地排序,返回 None)
nums = [3, 1, 4, 1, 5, 9, 2, 6]
nums.sort()
print(nums)              # [1, 1, 2, 3, 4, 5, 6, 9]

nums.sort(reverse=True)  # 降序
print(nums)              # [9, 6, 5, 4, 3, 2, 1, 1]

# 逐行解剖
# ① sort() 原地排序,返回 None
# ② 警告:不要写 result = nums.sort() → result 是 None!
```

```python
# 示例 2:查询 —— index / count / in
nums = [3, 1, 4, 1, 5, 9, 2, 6]
print(nums.index(4))     # 2
print(nums.count(1))     # 2
print(5 in nums)         # True
print(7 in nums)         # False
```

**常见错误**

> 1. **sort 返回 None**:
>    `sorted_nums = nums.sort()` → sorted_nums 是 None!
>    正确做法:先 `nums.sort()`,再 `print(nums)`
> 2. **index 找不到值**:
>    `nums.index(99)` → ValueError

> **问自己**:
> - `sort()` 和 `sorted()` 有什么区别?
> - 如果列表里有重复值,index 返回哪个?
> - `in` 运算符对列表和字符串都适用吗?

#### 列表遍历 —— for + enumerate

> **解释**

用 `for` 循环遍历列表,可以逐个访问元素。
`enumerate(lst)` 同时拿到索引和值。

```python
# 示例 1:基本遍历
fruits = ["苹果", "香蕉", "橘子"]
for fruit in fruits:
    print(fruit)

# 示例 2:enumerate 同时拿索引和值
for index, fruit in enumerate(fruits):
    print(f"索引 {index}: {fruit}")

# 逐行解剖
# ① for fruit in fruits 逐个访问元素
# ② enumerate(fruits) 返回 (索引, 元素) 元组
```

#### 字典创建与取值 —— 带 key 的快递柜

> **痛点**

列表用数字索引(0, 1, 2)访问,
但现实中数据通常有名字。
比如学生信息:姓名、年龄、城市 ——
用列表存,还得记住"索引 0 是姓名",容易出错。

> **类比**

**字典** = 带标签的快递柜。
你只要说"取 key 为 '姓名' 的柜子",就能拿到值。
不需要记住"第几号柜子"。

> **解释**

字典用**花括号** `{}` 表示,
元素是 **key: value** 对。
key 必须是不可变类型(字符串、数字),value 任意。

**ASCII 内存图 —— 字典的存储结构**

```
stu = {"name": "小明", "age": 18, "city": "成都"}

内存中:
  变量 stu ──→  ┌─────────┬─────────┐
                │ key     │ value   │
                ├─────────┼─────────┤
                │ "name"  │ "小明"  │
                │ "age"   │ 18      │
                │ "city"  │ "成都"  │
                └─────────┴─────────┘
取值: stu["name"] → "小明"(通过 key 找 value)
```

```python
# 示例 1:创建字典,用 key 取值
stu = {"name": "小明", "age": 18, "city": "成都"}
print(stu["name"])    # 小明
print(stu["age"])     # 18
# print(stu["phone"]) # KeyError! key 不存在

# 逐行解剖
# ① stu["name"] 在字典中查找 key "name"
# ② 找到 → 返回对应的 value "小明"
# ③ 找不到 → KeyError
```

```python
# 示例 2:安全取值 —— get() 带默认值
stu = {"name": "小明", "age": 18}
print(stu.get("name", "未知"))      # 小明
print(stu.get("phone", "未填写"))   # 未填写(不报错)

# 逐行解剖
# ① get("name", "未知") 找到 → 返回 "小明"
# ② get("phone", "未填写") 找不到 → 返回默认值
```

> **BREAK IT:对不存在的 key 用 [] 取值**

```python
# ============ BREAK IT 演示 ============
# stu = {"name": "小明"}
# print(stu["phone"])   # KeyError: 'phone'
# ============ END BREAK IT ============

# 原因:[] 取值,key 不存在就报错
# 修正:用 stu.get("phone", "默认值")
```

**常见错误**

> 1. **用不存在的 key 直接取值**:
>    `stu["phone"]` → KeyError。
>    建议用 `stu.get("phone", "默认值")`
> 2. **key 大小写敏感**:
>    `stu["Name"]` 和 `stu["name"]` 是不同的 key

> **问自己**:
> - 列表用索引取值,字典用什么取值?
> - `d[key]` 和 `d.get(key)` 什么时候结果不同?
> - 字典的 key 可以是列表吗?为什么?

#### 字典 CRUD —— 增 / 改 / 删

> **痛点**

创建字典后,经常需要动态添加新字段、
修改已有字段、删除不需要的字段。

> **类比**

字典是一个**快递柜系统**:
"增" = 新开一个柜子,放入东西;
"改" = 打开已有柜子,换里面的东西;
"删" = 拆掉一个柜子。

```python
# 示例:增 / 改 / 删 综合演示
stu = {"name": "小明", "age": 18}

# 增:给新 key 赋值
stu["city"] = "成都"
print(stu)    # {'name': '小明', 'age': 18, 'city': '成都'}

# 改:给已有 key 赋值(覆盖)
stu["age"] = 19
print(stu)    # {'name': '小明', 'age': 19, 'city': '成都'}

# 删:pop 删除 key,返回 value
removed = stu.pop("city")
print(removed)    # 成都
print(stu)        # {'name': '小明', 'age': 19}

# 逐行解剖
# ① stu["city"] = "成都" → key 不存在 → 新增
# ② stu["age"] = 19 → key 存在 → 覆盖
# ③ pop("city") → 删除并返回 value
```

**常见错误**

> 1. **pop 不存在的 key**:
>    `stu.pop("phone")` → KeyError
> 2. **以为修改 key 会新增**:
>    如果 key 已存在,赋值是**修改**,不是新增

> **问自己**:
> - 如何判断一个 key 在字典中是"新增"还是"修改"?
> - `pop()` 和 `del d[key]` 有什么区别?
> - 字典的 key 可以重复吗?

#### 字典遍历 —— keys / values / items

> **痛点**

拿到一个字典,经常需要:
打印所有 key、统计所有 value、同时看 key 和 value。
用 `for key in dict` 只能拿到 key,怎么同时拿 value?

> **类比**

字典是一排带标签的快递柜:
`keys()` = 只看所有标签,
`values()` = 只看所有柜子里的东西,
`items()` = 同时看标签和东西(最常用)。

> **解释**

`items()` 是字典**最强的迭代方式** ——
同时拿到 key 和 value,一次遍历搞定。

```python
# 示例 1:三种遍历方式
stu = {"name": "小明", "age": 18, "city": "成都"}

# 方式 1:遍历 key(最基础)
for key in stu:
    print(key)

# 方式 2:遍历 value
for value in stu.values():
    print(value)

# 方式 3:同时遍历 key 和 value(最强)
for key, value in stu.items():
    print(f"{key}: {value}")

# 逐行解剖
# ① for key in stu 等价于 for key in stu.keys()
# ② stu.values() 返回所有 value
# ③ stu.items() 返回 (key, value) 元组,解包给 key, value
```

```python
# 示例 2:用 items() 做条件筛选
scores = {"张三": 85, "李四": 92, "王五": 78}
for name, score in scores.items():
    if score >= 90:
        print(f"{name} 优秀")

# 逐行解剖
# ① items() 返回 (key, value) 元组
# ② 每次迭代解包: name="张三", score=85
# ③ score >= 90? → 满足才打印
```

**常见错误**

> 1. **遍历字典时修改字典**:
>    在 `for k in d` 循环里 `d.pop(k)` 会报错
> 2. **以为 items() 返回列表**:
>    它返回的是**视图对象**,需要 `list(d.items())` 转列表

> **问自己**:
> - `for key in d` 和 `for key in d.keys()` 有什么区别?
> - 为什么 `items()` 是最强的迭代方式?
> - 能在遍历字典的同时添加/删除 key 吗?

#### 综合练习 —— 通讯录 v1

现在把**列表和字典**串成一个小项目。

**需求清单**:
1. 用**字典**存储通讯录,key = 姓名, value = 电话
2. 用**菜单循环**让用户选择操作
3. 功能:添加、查找、删除、查看全部、退出

```python
# 示例:通讯录 v1 完整实现
contacts = {}

while True:
    print("\n=== 通讯录 ===")
    print("1.添加  2.查找  3.删除  4.全部  0.退出")
    choice = input("请选择:")
    if choice == "1":
        name = input("姓名:")
        phone = input("电话:")
        contacts[name] = phone
        print("添加成功")
    elif choice == "2":
        name = input("查找姓名:")
        print(contacts.get(name, "未找到"))
    elif choice == "3":
        name = input("删除姓名:")
        if name in contacts:
            contacts.pop(name)
            print("删除成功")
        else:
            print("查无此人")
    elif choice == "4":
        if not contacts:
            print("通讯录为空")
        else:
            for name, phone in contacts.items():
                print(f"{name}: {phone}")
    elif choice == "0":
        print("再见!")
        break
    else:
        print("无效选择")

# 逐行解剖
# ① contacts = {} 创建空字典
# ② while True 无限循环,直到 break
# ③ contacts.get(name, "未找到") 安全查找
# ④ for name, phone in contacts.items() 遍历
```

#### 今日小结

| 概念 | 解决的痛点 | 关键语法 |
| --- | --- | --- |
| 列表创建与索引切片 | 把一排数据粘在一起 | `[]` / `[i]` / `[a:b]` |
| 列表可变性 | 直接修改元素不报错 | `lst[i] = x` |
| 列表 CRUD(增删) | 动态添加/删除元素 | `append` / `pop` / `remove` |
| 列表排序与查询 | 排序、查找、计数 | `sort` / `index` / `count` / `in` |
| 字典创建与取值 | 用 key 代替数字索引 | `{}` / `d[key]` / `d.get()` |
| 字典 CRUD | 增/改/删 key-value | `d[k]=v` / `d.pop(k)` |
| 字典遍历 | 同时拿到 key 和 value | `items()` / `keys()` / `values()` |
