#### 比较运算符 —— 判断两个值的关系

> **痛点**

程序需要判断"年龄是否大于 18"、"密码是否正确",
但之前学的 `=` 只能赋值,没法做比较。

> **类比**

比较运算符就像天平:
`>` 看哪边重,`==` 看两边是否一样重,
`!=` 看两边是不是不一样。

> **解释**

比较运算符对两个值做比较,
结果为 **True**(真)或 **False**(假),
这两种值叫做**布尔值**(bool)。

**ASCII 内存图 —— 比较运算的结果**

```
print(3 > 1)

计算过程:
  ① 取出 3 和 1
  ② 比较:3 大于 1 吗? → 是
  ③ 返回 True(布尔值)

内存:
  变量 a = True  ← 比较结果存为布尔值
```

**关键**:比较运算的结果是 **bool** 类型,
可以直接赋值给变量,也可以用于 if 判断。

```python
# 示例 1:六种比较运算符
print(3 > 1)     # True —— 大于
print(3 < 1)     # False —— 小于
print(3 == 3)    # True —— 双等号:比较相等
print(3 != 4)    # True —— 不等
print(3 >= 3)    # True —— 大于等于
print(3 <= 2)    # False —— 小于等于

# 逐行解剖
# ① 3 > 1 比较 3 是否大于 1 → True
# ② 3 == 3 双等号是比较,不是赋值
# ③ != 是不等,>= 是大于等于
```

```python
# 示例 2:比较结果存入变量
age = 18
is_adult = age >= 18    # 比较结果赋值给变量
print(is_adult)         # True
print(type(is_adult))   # <class 'bool'>
```

**常见错误**

> 1. **错误现象**:`if age = 18:` 报错 SyntaxError
>    **原因:**条件里用了单等号(赋值),应该用 `==`
> 2. **错误现象**:`print(3 > 1 > 0)` 结果出乎意料
>    **原因:**Python 支持链式比较,但可读性差,
>    建议拆成 `3 > 1 and 1 > 0`

> - 比较运算符的结果是什么类型?
> - `3 >= 3` 是 True 还是 False?为什么?

#### if 基础 —— 单向判断(如果...就...)

> **痛点**

程序需要"满足某条件时才做某事",
比如"满 18 岁才显示成人内容",
没有 if,只能对所有用户显示相同内容。

> **类比**

小区门禁:**如果**你是业主,**就**开门;
不是业主?什么也不做,直接走开。
这就是"单向判断" —— 只处理成立的情况。

> **解释**

`if` 后面跟一个条件表达式,
条件为 True 时执行缩进块内的代码,
条件为 False 时跳过整个缩进块。

**ASCII 内存图 —— if 的执行流程**

```
age = 20
if age >= 18:
    print("已成年")     ← 条件为 True,执行
print("程序结束")       ← 无论条件如何,都执行

执行流程:
  ① age = 20
  ② 判断 age >= 18 → True
  ③ 进入缩进块 → 打印"已成年"
  ④ 跳出 if → 打印"程序结束"

如果 age = 15:
  ② 判断 age >= 18 → False
  ③ 跳过缩进块
  ④ 直接打印"程序结束"
```

**关键**:Python 用**缩进**(4 个空格)表示代码块,
不需要像 C/Java 那样写花括号。

```python
# 示例 1:最简单的 if
age = 20
if age >= 18:
    print("你已经成年了")      # 缩进 4 格
    print("可以考取驾照")      # 同一代码块
print("这行始终执行")          # 没有缩进,不属于 if

# 逐行解剖
# ① if age >= 18: if + 条件 + 冒号
# ② print("你已经成年了") 缩进 4 格,属于 if 块
# ③ print("这行始终执行") 无缩进,不属于 if
```

**常见错误**

> 1. **错误现象**:忘记写冒号
>    **原因:**`if age >= 18` → SyntaxError。修正:`if age >= 18:`
> 2. **错误现象**:缩进不一致
>    **原因:**混用 Tab 和空格 → IndentationError

> **问自己**:
> - if 末尾为什么要加冒号?
> - 缩进 2 个空格可以吗?8 个呢?
> - 如果条件是 False,if 块里的代码会执行吗?

#### if-else —— 双向判断(如果...否则...)

> **痛点**

if 只能处理"成立"的情况,
但很多时候"不成立"时也要做点什么,
比如"成年显示 A,未成年显示 B"。

> **类比**

红绿灯:**如果是绿灯,就走;否则,就停。**
两条路必走其一,没有第三种可能。

> **解释**

`if-else` 让程序二选一:
条件成立执行 if 块,不成立执行 else 块,
**有且只有一条路**会被执行。

**ASCII 内存图 —— if-else 的执行流程**

```
age = 15
if age >= 18:
    print("成年人")     ← 条件为 True 时走这里
else:
    print("未成年人")   ← 条件为 False 时走这里
print("判断结束")       ← 无论走哪条,都会到这里

执行流程(age = 15):
  ① 判断 age >= 18 → False
  ② 跳过 if 块
  ③ 进入 else 块 → 打印"未成年人"
  ④ 跳出 if-else → 打印"判断结束"
```

**关键**:if 和 else 是**互斥**的,
执行了 if 就不会执行 else,反之亦然。

```python
# 示例:if-else 基本用法
age = 15
if age >= 18:
    print("成年人")
else:
    print("未成年人")
print("判断结束")

# 逐行解剖
# ① if age >= 18: 判断条件
# ② print("成年人") 条件为 True 时执行
# ③ else: 不需要条件,表示"其他所有情况"
# ④ print("未成年人") 条件为 False 时执行
```

> **BREAK IT:else 后面写条件会怎样**

```python
# ============ BREAK IT 演示 ============
# age = 20
# if age >= 18:
#     print("成年人")
# else age < 18:       #  SyntaxError! else 不能带条件
#     print("未成年人")
# ============ END BREAK IT ============
```

**常见错误**

> 1. **错误现象**:else 后面写了条件
>    **原因:**`else age < 18:` → SyntaxError。修正:`else:`
> 2. **错误现象**:if 和 else 缩进不对齐
>    **原因:**缩进不一致 → IndentationError

> **问自己**:
> - if 和 else 能同时执行吗?
> - else 后面能加条件吗?
> - 如果 if 条件成立,else 块会执行吗?

#### if-elif-else —— 多向判断(多选一)

> **痛点**

if-else 只能二选一,
但现实中经常需要多个分支:
成绩分 A/B/C/D/F,年龄分老/中/青/少。

> **类比**

考试评分:**90 分以上 → A,80-89 → B,
70-79 → C,60-69 → D,60 以下 → F。**
从上到下,第一个满足的就"命中",然后跳出。

> **解释**

`elif` 是 **else if** 的缩写,
可以写多个,Python 从上到下依次判断,
**第一个为 True 的分支执行后,整个结构结束**。

**ASCII 内存图 —— if-elif-else 的执行流程**

```
score = 85
if score >= 90:
    print("A")         ← 85 >= 90? 否,跳过
elif score >= 80:
    print("B")         ← 85 >= 80? 是,执行,跳出
elif score >= 70:
    print("C")         ← 不再判断
else:
    print("F")         ← 不再判断

执行流程:
  ① 判断 score >= 90 → False
  ② 判断 score >= 80 → True → 打印 B
  ③ 跳出整个结构,不再判断后续条件
```

**关键**:elif 是**顺序敏感**的,
大范围条件要写在小范围前面,否则会被提前拦截。

```python
# 示例 1:成绩等级判断
score = 85
if score >= 90:
    print("A")
elif score >= 80:
    print("B")         # 命中,跳出
elif score >= 70:
    print("C")         # 不再判断
elif score >= 60:
    print("D")
else:
    print("F")

# 逐行解剖
# ① if score >= 90: 第一个条件
# ② elif score >= 80: 隐含条件:90 > score >= 80
# ③ elif score >= 70: 隐含条件:80 > score >= 70
# ④ else: 兜底,以上都不满足时执行
```

> **BREAK IT:顺序写反会怎样**

```python
# ============ BREAK IT 演示 ============
# score = 95
# if score >= 60:       # 先写宽松条件
#     print("D")        # 95 >= 60,命中,跳出
# elif score >= 90:     # 永远不会执行!
#     print("A")
# else:
#     print("F")
# ============ END BREAK IT ============
```

**要点**:每个 `elif` 都隐含了上面所有条件**不成立**,
所以顺序很重要,从严格到宽松。

**常见错误**

> 1. **错误现象**:顺序写反,大范围永远执行不到
>    **原因:**`if score >= 60:` 先写宽松条件,`>= 90` 永远不会执行
> 2. **错误现象**:忘了 elif 需要条件
>    **原因:**`elif:` → SyntaxError。修正:`elif score >= 80:`

> **问自己**:
> - elif 和 else 有什么区别?
> - 如果多个条件都为 True,会执行几个分支?
> - 为什么顺序很重要?能举例说明吗?

#### 逻辑运算符 —— 组合多个条件

> **痛点**

现实中的条件往往不是单一的:
"年龄 >= 18 **且** 有身份证"才能开户,
"是会员 **或** 消费满 100"才能打折。
需要把多个条件**组合**在一起。

> **类比**

- **and**(且):考试两科都及格才通过
- **or**(或):有学生证或身份证就能入场
- **not**(非):不是会员就不能打折

> **解释**

逻辑运算符把多个布尔条件组合,
返回一个新的布尔值:
- `and`:两边都 True 才 True
- `or`:一边为 True 就 True
- `not`:取反,True 变 False

```python
# 示例 1:and —— 两边都满足才成立
age = 25
has_id = True
if age >= 18 and has_id:
    print("可以开户")
else:
    print("无法开户")

# 逐行解剖
# ① age >= 18 → True
# ② has_id → True
# ③ True and True → True → 进入 if 块
```

```python
# 示例 2:or —— 任意一个满足就成立
is_member = False
total_price = 150
if is_member or total_price >= 100:
    print("享受折扣")
else:
    print("原价")

# 逐行解剖
# ① is_member → False
# ② total_price >= 100 → True
# ③ False or True → True → 进入 if 块
```

```python
# 示例 3:not —— 取反
is_raining = False
if not is_raining:
    print("今天不下雨,可以出门")
else:
    print("今天下雨,带伞")
```

> **BREAK IT:and 两边不是完整表达式**

```python
# ============ BREAK IT 演示 ============
# age = 25
# if age >= 18 and <= 60:   #  SyntaxError! 右边不完整
#     print("劳动年龄")
# ============ END BREAK IT ============
```

**常见错误**

> 1. **错误现象**:and 两边不是完整表达式
>    **原因:**`if age >= 18 and <= 60:` → SyntaxError。修正:`if age >= 18 and age <= 60:`
> 2. **错误现象**:混淆 and 和 or
>    **原因:**`if age >= 18 or age <= 60:` 永远为 True。修正:用 `and`

> **问自己**:
> - `and` 和 `or` 的区别是什么?
> - `not True` 的结果是什么?
> - 为什么 `age >= 18 or age <= 60` 永远为 True?

#### 嵌套条件 —— if 里面再写 if

> **痛点**

有些判断需要**分步骤**:
先看"是否成年",再看"是否退休",
外层不满足时内层根本不用看。

> **类比**

入职流程:**先**看年龄是否达标,
**再**看学历是否达标,
**最后**看经验是否达标。
每一层都通过,才能进入下一层。

> **解释**

在一个 if 的缩进块内部再写一个 if,
就是**嵌套条件**。
内层 if 只有在外层条件为 True 时才会被判断。

**ASCII 内存图 —— 嵌套条件的执行流程**

```
age = 25
if age >= 18:                    ← 外层判断
    print("成年人")
    if age >= 60:                ← 内层判断
        print("老年人")
    else:
        print("劳动年龄人口")
else:
    print("未成年人")

执行流程(age = 25):
  ① 判断 age >= 18 → True
  ② 进入外层 if 块 → 打印"成年人"
  ③ 判断 age >= 60 → False
  ④ 进入内层 else → 打印"劳动年龄人口"
  ⑤ 跳出整个结构
```

**关键**:嵌套不宜超过 2 层,
超过建议用 `and` 扁平化。

```python
# 示例 1:嵌套条件 —— 判断年龄阶段
age = 25
if age >= 18:
    print("成年人")
    if age >= 60:
        print("老年人")
    else:
        print("劳动年龄人口")
else:
    print("未成年人")

# 逐行解剖
# ① if age >= 18: 外层判断
# ② print("成年人") 外层为 True 时执行
# ③ if age >= 60: 内层判断(只有外层为 True 才判断)
# ④ else: 内层 else
```

```python
# 示例 2:用 and 扁平化(推荐)
age = 25
if age >= 60:
    print("老年人")
elif age >= 18:
    print("劳动年龄人口")
else:
    print("未成年人")
```

**常见错误**

> 1. **错误现象**:内层 if 和外层 if 缩进相同
>    **原因:**内层 if 要比外层多缩 4 格
> 2. **错误现象**:嵌套超过 3 层,代码难以阅读
>    **原因:**建议用 elif + and 扁平化

> **问自己**:
> - 嵌套 if 的内层条件什么时候才会被判断?
> - 嵌套和扁平化(and/elif)哪个更好?
> - 什么情况下必须用嵌套?

#### 三元表达式 —— 一行搞定简单二选一

> **痛点**

简单的 if-else 要写 4 行,
如果只是"根据条件选一个值",
能不能更简洁?

> **类比**

点餐:**如果**有会员价,**就**按会员价;
**否则**按原价。
一句话就能说清,不需要写一大段。

> **解释**

三元表达式语法:**值1 if 条件 else 值2**
条件成立返回值1,不成立返回值2。
只适合**简单赋值**,复杂逻辑不要强塞一行。

```python
# 示例 1:三元表达式基本用法
age = 20
status = "成年" if age >= 18 else "未成年"
print(status)    # 成年

# 逐行解剖
# ① "成年" if age >= 18 else "未成年" 三元表达式
# ② 先判断 age >= 18
# ③ 为 True → 返回 "成年"
# ④ 为 False → 返回 "未成年"
# ⑤ 把返回值赋给 status
```

```python
# 示例 2:三元表达式的典型场景 —— 求绝对值
num = -5
abs_value = num if num >= 0 else -num
print(abs_value)    # 5
```

**常见错误**

> 1. **错误现象**:把 if-else 语句塞进一行
>    **原因:**`if age >= 18: print("A") else: print("B")` PEP 8 不推荐
> 2. **错误现象**:嵌套三元表达式
>    **原因:**可读性极差,建议用 if-elif-else

> **问自己**:
> - 三元表达式适合什么场景?
> - 三元表达式能替代所有 if-else 吗?
> - 什么时候不该用三元表达式?

#### in 运算符 —— 判断元素是否在序列中

> **痛点**

需要判断"某个值是否在列表中",
或"某个字符是否在字符串中",
用多个 `or` 写起来很繁琐。

> **类比**

点名:**"张三"在不在名单里?**
翻名单找一遍,找到了就说"在",
没找到就说"不在"。
`in` 就是帮你"翻名单"的运算符。

> **解释**

`in` 判断元素是否存在于
字符串、列表、元组等**可迭代对象**中,
返回 True / False。
`not in` 是反向判断(不在其中)。

```python
# 示例 1:in 判断字符串中是否包含子串
email = "zhangsan@example.com"
if "@" in email:
    print("看起来像邮箱")
else:
    print("不是邮箱格式")

# 逐行解剖
# ① "@" in email 判断 @ 是否在 email 中
# ② 返回 True → 进入 if 块
```

```python
# 示例 2:in 判断列表中是否有某个元素
fruits = ["苹果", "香蕉", "橘子"]
if "苹果" in fruits:
    print("有苹果")
else:
    print("没苹果")
```

**常见错误**

> 1. **错误现象**:对数字使用 in
>    **原因:**`if 3 in 123:` → TypeError。修正:`if "3" in "123":`
> 2. **错误现象**:混淆 in 和 ==
>    **原因:**`in` 是子串匹配,`==` 是完全相等

> **问自己**:
> - `in` 可以用于哪些数据类型?
> - `in` 和 `==` 有什么区别?
> - `not in` 等价于什么?

#### 综合练习 1 —— 成绩等级判断

把今天学的**比较运算符 + if-elif-else**串成一个完整程序。

**需求**:
- 输入成绩(0-100)
- 输出等级:A(90-100) / B(80-89) /
  C(70-79) / D(60-69) / F(<60)
- 输入超出范围时提示"成绩无效"

```python
# 综合练习 1:成绩等级判断
score = 85

# 先判断是否合法
if score < 0 or score > 100:
    print("成绩无效")
elif score >= 90:
    print("A")
elif score >= 80:
    print("B")
elif score >= 70:
    print("C")
elif score >= 60:
    print("D")
else:
    print("F")

# 逐行解剖
# ① score < 0 or score > 100 先排除非法输入
# ② score >= 90 最高等级先判断
# ③ 依次向下,隐含条件自动排除已判断过的
# ④ else 兜底,处理 < 60 的情况
```

#### 综合练习 2 —— BMI 计算器

把**if-else + 比较运算符 + 逻辑运算**综合运用。

**需求**:
- 输入身高(m)和体重(kg)
- 计算 BMI = 体重 / 身高^2
- 根据中国标准输出偏瘦/正常/偏胖/肥胖
- BMI < 18.5 偏瘦,18.5-24 正常,
  24-28 偏胖,>= 28 肥胖

```python
# 综合练习 2:BMI 计算器
height = 1.75   # 身高(米)
weight = 70     # 体重(公斤)

bmi = weight / (height ** 2)
print(f"BMI = {bmi:.1f}")

if bmi < 18.5:
    print("偏瘦")
elif bmi < 24:
    print("正常")
elif bmi < 28:
    print("偏胖")
else:
    print("肥胖")

# 逐行解剖
# ① height ** 2 计算身高的平方
# ② weight / (height ** 2) 计算 BMI
# ③ f"{bmi:.1f}" 保留 1 位小数
# ④ 按 BMI 区间判断体型
```
