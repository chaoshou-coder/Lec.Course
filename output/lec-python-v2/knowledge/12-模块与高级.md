#### 模块导入 import —— 代码的"抽屉"

> **痛点**

一个 .py 写到 1000 行,翻来翻去找函数非常痛苦,
多个函数同名还容易冲突。

> **类比**

模块就像抽屉 —— 把不同工具分门别类放进各自的抽屉,
用到哪个拉开哪个,不需要全堆在桌面上。

> **解释**

Python 把每个 .py 文件看作一个**模块**,
用 import 即可把另一个文件的函数/变量/类引入当前文件。
三种写法:
- `import 模块名` → 通过 `模块名.成员` 访问
- `from 模块名 import 成员` → 直接用成员名,无需前缀
- `import 模块名 as 别名` → 给长名字起短名

**ASCII 内存图 —— 命名空间**

```
当前文件命名空间            math 模块内部
+--------------+          +----------------+
| sqrt --------+--------->| sqrt 函数      |
| pi  ---------+--------->| pi = 3.14159.. |
+--------------+          +----------------+

from math import sqrt 之后:
当前命名空间里多了一个 sqrt,
调用时直接写 sqrt(16),不再需要 math 前缀
```

**关键**:模块只加载一次,重复 import 不会重新执行。

```python
# 示例 1:import 全量导入,通过 模块名.成员 访问
import math

result = math.sqrt(16)   # 通过 math. 前缀访问
pi = math.pi
print(result)            # 4.0
print(pi)                # 3.141592653589793
```

**逐行解剖**

① `import math` 在 sys.path 中查找 math 模块
② 首次导入时执行模块代码,创建模块对象
③ `math.sqrt(16)` 从模块中找到 sqrt 函数,调用返回 4.0
④ 重复 import 不会重新执行模块,模块只加载一次

```python
# 示例 2:from ... import 按需导入,直接用名字
from math import sqrt, pi

print(sqrt(25))          # 5.0(不需要 math. 前缀)
print(pi)                # 3.141592653589793
```

**逐行解剖**

① `from math import sqrt, pi` 从模块取出 sqrt 和 pi
② 把这两个名字绑定到当前命名空间
③ `sqrt(25)` 在当前命名空间找到 sqrt,调用返回 5.0

```python
# 示例 3:import as 给模块起别名
import math as m
from math import factorial as fac

print(m.sqrt(9))         # 3.0
print(fac(5))            # 120
```

**常见错误**

> 1. **错误现象**:`ImportError: No module named mymodule`
>    **原因**:mymodule.py 不在当前目录或 sys.path 中的任何路径
> 2. **错误现象**:`NameError: name x is not defined`
>    **原因**:用了 `from x import y`,但 x 模块里根本没有 y 这个成员

#### 生成器 yield —— 按需点餐

> **痛点**

有 100 万个数据要处理,一次性读到列表里会撑爆内存。

> **类比**

生成器就像按需点餐 —— 吃一盘上一盘,
不像列表那样把菜全端上来。

> **解释**

包含 yield 的函数就是**生成器函数**,
调用它不会立即执行函数体,而是返回一个生成器对象。
每次 next() 或在 for 循环中,函数执行到 yield 就暂停
并返回一个值,下次再从暂停处继续。

**ASCII 内存图 —— 生成器的暂停与恢复**

```
gen = squares(4)

调用 squares(4):
  ① 不执行函数体
  ② 返回生成器对象 ──→ gen

第 1 次 next(gen):
  ① 进入函数, i=0
  ② yield 0 → 暂停,返回 0
      ┌──────────────┐
      │ 暂停在 yield  │ ← 状态冻结
      │ i=0          │
      └──────────────┘

第 2 次 next(gen):
  ① 从暂停处继续, i=1
  ② yield 1 → 暂停,返回 1
```

**关键**:生成器只遍历一次,遍历完就空了。

```python
# 示例 1:生成器函数用 yield 逐个产出平方数
def squares(n):
    for i in range(n):
        yield i * i       # 暂停并产出 i 的平方

gen = squares(4)
print(gen)               # <generator object squares at 0x...>

for v in squares(4):
    print(v)             # 0  1  4  9
```

**逐行解剖**

① `def squares(n):` 定义生成器函数(包含 yield)
② `gen = squares(4)` 不执行函数体,只返回生成器对象
③ for 第 1 次迭代:进入函数 i=0,遇到 yield 0,暂停返回 0
④ for 第 2 次迭代:从暂停处继续 i=1,yield 1,暂停返回 1
⑤ 循环耗尽后自动抛出 StopIteration,for 结束

```python
# 示例 2:手动用 next() 推进生成器
def fib(n):
    a, b = 0, 1
    for _ in range(n):
        yield a          # 产出当前值
        a, b = b, a + b  # 同步更新

gen = fib(5)
print(next(gen))         # 0
print(next(gen))         # 1
print(next(gen))         # 1
```

**常见错误**

> 1. **错误现象**:`TypeError: object of type generator has no len()`
>    **原因**:生成器不能 len(),因为它不一次性生成所有值
> 2. **错误现象**:第二次 for 遍历生成器结果为空
>    **原因**:生成器只能遍历一次,遍历完就空了

#### 装饰器 @decorator —— 包装礼物

> **痛点**

想给多个函数加计时逻辑,每个函数手动改就是复制粘贴。

> **类比**

装饰器就像包装礼物 —— 不改变礼物本身,
在外面多包一层包装纸。

> **解释**

装饰器本质是**接收函数、返回新函数**的高阶函数。
`@decorator` 语法糖等价于 `func = decorator(func)`。
`@functools.wraps(fn)` 保留原函数的 `__name__` 等元信息。

**ASCII 内存图 —— 装饰器的执行过程**

```
@timer
def slow_sum(n):
    ...

等价于: slow_sum = timer(slow_sum)

① 调用 timer(slow_sum)
② 返回 wrapper 函数
③ 变量 slow_sum 现在指向 wrapper

调用 slow_sum(100):
  ① 实际调用 wrapper(100)
  ② 记录 start 时间
  ③ 调用原 slow_sum(100) → 得到结果
  ④ 计算耗时并打印
  ⑤ 返回结果
```

**关键**:装饰器在不修改原函数的前提下增加额外功能。

```python
# 示例 1:给函数加计时的装饰器
import time
import functools

def timer(fn):
    @functools.wraps(fn)     # 保留原函数的 __name__ 等元信息
    def wrapper(*args, **kwargs):
        start = time.time()
        result = fn(*args, **kwargs)   # 调用原函数
        elapsed = time.time() - start
        print(f"{fn.__name__} 耗时 {elapsed:.4f} 秒")
        return result
    return wrapper

@timer
def slow_sum(n):
    total = 0
    for i in range(n):
        total += i
    return total

print(slow_sum(1000000))
```

**逐行解剖**

① `def timer(fn):` 装饰器函数,接收被装饰的函数
② `@functools.wraps(fn)` 把原函数的元信息复制到 wrapper
③ `def wrapper(*args, **kwargs):` 适配任意参数签名
④ `@timer` 语法糖,等价于 `slow_sum = timer(slow_sum)`
⑤ 调用 slow_sum 时实际调用 wrapper,计时后返回结果

```python
# 示例 2:演示 @wraps 的重要性
import functools

def without_wraps(fn):
    def wrapper(*args, **kwargs):
        return fn(*args, **kwargs)
    return wrapper

def with_wraps(fn):
    @functools.wraps(fn)
    def wrapper(*args, **kwargs):
        return fn(*args, **kwargs)
    return wrapper

@without_wraps
def greet1():
    pass

@with_wraps
def greet2():
    pass

print(greet1.__name__)    # wrapper (元信息丢失!)
print(greet2.__name__)    # greet2 (保留)
```

**常见错误**

> 1. **错误现象**:`TypeError: wrapper() takes 0 positional arguments`
>    **原因**:wrapper 参数签名写死,没写 `*args, **kwargs`
> 2. **错误现象**:调试时函数名全变成 wrapper
>    **原因**:忘记加 `@functools.wraps(fn)`,元信息丢失

#### 上下文管理器 with —— 自动关门

> **痛点**

打开文件后忘记 close(),资源泄漏;
异常发生时 close() 根本没执行。

> **类比**

上下文管理器就像自动门 —— 进入时自动开,
离开时自动关,哪怕里面发生火灾(异常)也会关。

> **解释**

`with` 语句调用对象的 `__enter__` 和 `__exit__` 方法:
- `__enter__`:进入代码块时执行(获取资源)
- `__exit__`:离开代码块时执行(释放资源,即使有异常)

**ASCII 内存图 —— with 的执行流程**

```
with open("data.txt") as f:
    data = f.read()
    ┌─────────────────────────────┐
    │ with 代码块                  │
    │                             │
    │  ① __enter__ → 返回 f      │
    │  ② 执行 f.read()            │
    │  ③ __exit__ → 自动 close   │
    │     (即使 read 抛异常也会关) │
    └─────────────────────────────┘
```

**关键**:with 保证资源一定被释放,不用手动 close()。

```python
# 示例 1:用 with 读写文件(最常用)
# 写入文件
with open("demo.txt", "w", encoding="utf-8") as f:
    f.write("Hello\nWorld")
# 离开 with 块,f 自动 close,不需要 f.close()

# 读取文件
with open("demo.txt", "r", encoding="utf-8") as f:
    content = f.read()
print(content)            # Hello\nWorld
```

**逐行解剖**

① `with open(...) as f:` 打开文件,`__enter__` 返回文件对象
② `f.write(...)` 在代码块内操作文件
③ 离开 with 块时自动调用 `__exit__`,关闭文件
④ 即使 write 抛异常,文件也会被关闭

```python
# 示例 2:自定义上下文管理器(类实现)
class Timer:
    """计时上下文管理器:统计代码块执行时间"""
    import time

    def __enter__(self):
        self.start = __import__("time").time()
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        elapsed = __import__("time").time() - self.start
        print(f"代码块耗时 {elapsed:.4f} 秒")
        return False  # False = 不吞掉异常,继续往外抛

# 使用自定义上下文管理器
with Timer():
    total = sum(range(100000))
    print(f"计算结果: {total}")
```

**常见错误**

> 1. **错误现象**:在 with 块外面还操作文件对象
>    **原因**:with 结束后文件已 close,再 read/write 报 ValueError
> 2. **错误现象**:`__exit__` 返回 True 导致异常被吞掉
>    **原因**:`__exit__` 返回 True 表示"已处理异常",通常应返回 False
