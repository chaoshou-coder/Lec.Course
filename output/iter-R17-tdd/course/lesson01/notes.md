### Day 01 · TDD 三步循环:红-绿-重构

> **痛点**:你写代码很快但 bug 也多。听说 TDD 能提升质量,但觉得"写测试太慢"。今天你将体验 TDD 的真实感受 —— 先写失败测试,再写代码通过测试。
> **类比**:TDD 就像"先写考试再教学生" —— 先定验收标准(测试),再教最少内容通过考试(代码),最后优化教学方法(重构)。
> **解释**:**TDD = 测试驱动开发**,三步循环:红(测试失败) → 绿(代码通过) → 重构(优化代码)。

---

#### 红 —— 先写失败测试

> **痛点**:你习惯先写代码再补测试,但补的测试往往流于形式(测的是实现,不是行为)。
> **类比:**红 = "考试还没考,先出卷子"。你不知道学生能考多少分,但你知道什么算及格。
> **解释:**先写测试,此时测试应该是失败的(因为还没写代码)。这验证了测试真的在测东西。

```python
# test_calculator.py —— 先写测试(此时测试会失败)

def test_add():
    assert add(2, 3) == 5

def test_subtract():
    assert subtract(5, 3) == 2

# 运行 pytest test_calculator.py
# 结果:FAILED —— NameError: name 'add' is not defined
# 这就是"红" —— 测试失败了!
```

**逐行解剖**

- `assert add(2, 3) == 5` = 断言 add(2, 3) 应该等于 5
- `NameError` = 因为 add 函数还没定义,所以测试失败
- 红 = 测试失败 = 验证测试真的在测东西

---

#### 绿 —— 写最少代码通过测试

> **痛点**:你可能会一次性写完所有功能。TDD 要求:只写能通过当前测试的最少代码。
> **类比:**绿 = "教最少内容让学生及格"。不教多余的东西,只教能通过考试的。
   **解释:**写最少的代码让测试通过,不添加任何额外功能。

```python
# calculator.py —— 只写能通过测试的最少代码

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

# 再运行 pytest test_calculator.py
# 结果:PASSED —— 2 passed
# 这就是"绿" —— 测试通过了!
```

---

#### 重构 —— 优化代码

> **痛点**:代码能工作但可能不够好。TDD 的第三步是重构 —— 在测试保护下优化代码。
> **类比:**重构 = "优化教学方法"。学生已经及格了,现在让教学更高效。
> **解释:**测试通过 = 安全网。你可以放心重构(重命名/拆分/消除重复),测试会通过说明没破坏功能。

```python
# 重构示例:消除重复(如果有多处 a + b)
def add(a, b):
    """两个数相加"""
    return operator.add(a, b)  # 用标准库

# 重构后再次运行测试 —— 应该仍然通过
```

---

#### 执行过程跟踪

```python
# --- 执行过程 ---
# Step 1(红):写 test_add → 运行 → FAILED(NameError)
# Step 2(绿):写 add 函数 → 运行 → PASSED
# Step 3(重构):优化 add → 运行 → PASSED(验证重构没破坏)
# 循环:写下一个测试(如 test_subtract) → 红 → 绿 → 重构
```

---

#### 常见错误

> 1. **错误现象**:先写代码再写测试
>    **原因:**违反了 TDD 的核心原则。修正:先写测试
> 2. **错误现象**:一次写太多功能
>   **原因:**TDD 要求"最少代码通过测试"。修正:一次只测一个行为
> 3. **错误现象**:测试测的是实现细节而不是行为
>    **原因:**测试应该测"输入→输出",而不是"内部怎么实现"。修正:只测公开接口

---

#### 学员代码区

```python
# TODO:写 test_multiply(测试 multiply 函数)
# TODO:运行测试(应该是红色/失败)
# TODO:写 multiply 函数(最少代码)
# TODO:再运行测试(应该是绿色/通过)
```

---

#### 参考答案

```python
# test_calculator.py
def test_multiply():
    assert multiply(2, 3) == 6

# calculator.py
def multiply(a, b):
    return a * b
```
