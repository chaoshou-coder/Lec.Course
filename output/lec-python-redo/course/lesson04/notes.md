### Day 04 · 循环入门:让程序"不知疲倦"

> **痛点**:你想让程序重复做一件事(比如打印 100 行),复制粘贴 100 次太蠢。今天你将学会让程序"自动重复" —— 像跑步机一样,设定好就停不下来。
> **类比**:循环就像"跑步机" —— 设定速度和时间,程序就一圈一圈跑,直到你喊停。
> **解释**:**循环 = 重复执行一段代码**。今天学:while、for、range、break/continue、嵌套循环。

---

#### while 循环 —— 条件成立就继续

> **痛点**:你想让程序"一直问密码,直到猜对为止",不知道怎么写。
> **类比**:while 就像"旋转门" —— 条件成立就一直转,条件不成立就停。
> **解释**:`while 条件:` 条件成立就执行循环体,执行完再判断,直到条件不成立。

```python
# 打印 1 到 5
i = 1                 # ① 起始值
while i <= 5:         # ② 终止条件
    print(i)
    i = i + 1         # ③ 递增步进(否则死循环!)

# 死循环风险演示(别轻易运行!)
# while True:
#     print("停不下来")   # 没有步进,永远执行
```

**逐行解剖**

- `i = 1` = 起始值(循环变量初始值)
- `i <= 5` = 终止条件(条件不成立就退出)
- `i = i + 1` = 递增步进(让循环变量变化,最终让条件不成立)
- 三个要素缺一不可,否则死循环

> **ASCII 执行流程图**
> ```
> i = 1
>    │
>    ▼
> i <= 5 ? ──No──▶ 退出循环
>    │
>   Yes
>    ▼
> print(i)  →  i = i + 1
>    │
>    └──(回到判断)──┘
>
> 输出: 1 2 3 4 5
> ```

**常见错误**

> 1. **错误现象**:程序永远运行,按 Ctrl+C 才停
>    **原因:**忘记写步进 `i = i + 1`,条件永远成立。修正:加上步进
> 2. **错误现象**:差一错误(off-by-one)
>    **原因:**`i <= 5` 和 `i < 5` 差一个数。修正:想清楚边界

#### break 与 continue —— 循环的"刹车"和"跳过"

> **痛点**:你想在循环中"提前退出"或"跳过某一次",不知道怎么写。
> **类比**:break 就像"紧急刹车" —— 立刻退出循环;continue 就像"跳过这一圈" —— 直接进入下一次。
> **解释**:`break` 立刻退出整个循环;`continue` 跳过本次循环剩余代码,进入下一次。

```python
# break:找到第一个能被 7 整除的数就退出
i = 1
while i <= 100:
    if i % 7 == 0:
        print(f"找到了: {i}")
        break           # 立刻退出循环
    i += 1

# continue:只打印奇数(跳过偶数)
i = 0
while i < 10:
    i += 1
    if i % 2 == 0:
        continue        # 跳过本次,进入下一次循环
    print(i)            # 偶数时不执行这行
```

**逐行解剖**

- `break` = 立刻退出循环(不管条件还成不成立)
- `continue` = 跳过本次循环体剩余代码,回到条件判断
- `i += 1` = `i = i + 1` 的简写

> **ASCII break/continue 示意图**
> ```
> break:                continue:
> ┌─────────────┐       ┌─────────────┐
> │ 循环体       │       │ 循环体       │
> │ if 条件:     │       │ if 条件:     │
> │   break ──▶ 退出    │   continue ──┐
> │ ...          │       │ ... (跳过)   │
> └─────────────┘       └──────┬──────┘
>                              └──▶ 回到判断
> ```

**常见错误**

> 1. **错误现象**:`continue` 放在 `i += 1` 之后导致死循环
>    **原因:**continue 跳过后面的代码,包括步进。修正:步进放在 continue 前
> 2. **错误现象**:以为 break 跳出 if
>    **原因:**break 跳出的是循环,不是 if。if 没有 break

#### for 循环 —— 遍历序列

> **痛点**:你想遍历字符串或列表的每个元素,用 while 太麻烦。
> **类比**:for 就像"逐个检票" —— 把序列里每个元素挨个处理一遍。
> **解释**:`for item in 序列:` 依次取出序列中的每个元素,赋值给 item。

```python
# 遍历字符串
s = "Hello"
for ch in s:
    print(ch)           # H e l l o(每个字符一行)

# 遍历列表
fruits = ["apple", "banana", "orange"]
for fruit in fruits:
    print(fruit)

# 累加器:求 1 到 100 的和
total = 0               # 累加器初始化为 0
for i in range(1, 101): # range(1,101) = 1,2,...,100
    total += i          # 累加
print(total)            # 5050

# 累乘器:求 10 的阶乘
product = 1             # 累乘器初始化为 1
for i in range(1, 11):
    product *= i        # 累乘
print(product)          # 3628800
```

**逐行解剖**

- `for ch in s:` = 依次取出 s 中每个字符
- `range(1, 101)` = 生成 1 到 100 的序列(左闭右开)
- `total += i` = 累加器(初始 0,不断加)
- `product *= i` = 累乘器(初始 1,不断乘)

> **ASCII 遍历示意图**
> ```
> s = "Hello"
>
> 第 1 轮: ch = 'H'  → print('H')
> 第 2 轮: ch = 'e'  → print('e')
> 第 3 轮: ch = 'l'  → print('l')
> 第 4 轮: ch = 'l'  → print('l')
> 第 5 轮: ch = 'o'  → print('o')
> 序列遍历完,退出
> ```

**常见错误**

> 1. **错误现象**:累加器初始化为 1
>    **原因:**累加器初始必须是 0(加 0 不变)。累乘器初始是 1(乘 1 不变)
> 2. **错误现象**:`range(100)` 以为是 1 到 100
>    **原因:**`range(100)` 是 0 到 99。1 到 100 用 `range(1, 101)`

#### range() —— 生成数字序列

> **痛点**:你需要一个数字序列(比如 0 到 9),不想手写。
> **类比**:range 就像"数字生成器" —— 指定起点终点步长,自动生成。
> **解释**:`range(start, stop, step)` 生成 start 到 stop-1 的序列,步长 step。

```python
# range(stop):0 到 stop-1
for i in range(5):
    print(i)            # 0 1 2 3 4

# range(start, stop):start 到 stop-1
for i in range(2, 6):
    print(i)            # 2 3 4 5

# range(start, stop, step):带步长
for i in range(1, 10, 2):
    print(i)            # 1 3 5 7 9(奇数)

# 负数步长(倒着数)
for i in range(10, 0, -1):
    print(i)            # 10 9 8 ... 1

# end='' 参数:不换行输出
for i in range(5):
    print(i, end=" ")   # 0 1 2 3 4 (同一行)
print()                 # 最后换行
```

**逐行解剖**

- `range(5)` = 0,1,2,3,4(省略 start,默认 0)
- `range(2, 6)` = 2,3,4,5(左闭右开)
- `range(1, 10, 2)` = 1,3,5,7,9(步长 2)
- `range(10, 0, -1)` = 10,9,...,1(负数步长倒着数)
- `end=" "` = 把 print 默认的换行改成空格

> **ASCII range 示意图**
> ```
> range(2, 8, 2):
>
> 2  3  4  5  6  7  8
> ↑     ↑     ↑     ↑
│ 2     4     6    (不含)
>
> 结果: 2, 4, 6
> ```

**常见错误**

> 1. **错误现象**:`range(1, 10, -1)` 什么都没输出
>    **原因:**start < stop 但 step 为负,走不到。修正:`range(10, 1, -1)`
> 2. **错误现象**:以为 range 返回列表
>    **原因:**range 返回的是 range 对象(省内存),不是列表

#### 嵌套循环 —— 循环里套循环

> **痛点**:你想打印九九乘法表或图形,一行一行打太麻烦。
> **类比**:嵌套循环就像"钟表" —— 秒针走一圈,分针走一格;内层循环完整执行一次,外层循环才进一步。
> **解释**:外层循环每执行一次,内层循环完整执行一圈。

```python
# 打印矩形(3 行 5 列星星)
for i in range(3):          # 外层:控制行
    for j in range(5):      # 内层:控制列
        print("*", end="")  # 不换行
    print()                 # 每行结束换行
# 输出:
# *****
# *****
# *****

# 九九乘法表
for i in range(1, 10):      # 外层:1 到 9
    for j in range(1, i+1): # 内层:1 到 i
        print(f"{j}×{i}={i*j}", end="\t")
    print()
```

**逐行解剖**

- 外层 `i` = 行数
- 内层 `j` = 列数
- `end=""` = 内层不换行
- 内层结束后 `print()` = 换行
- 九九乘法表:第 i 行有 i 个式子

> **ASCII 嵌套循环执行顺序**
> ```
> i = 1:  j = 1          → 1 个式子,换行
> i = 2:  j = 1, 2       → 2 个式子,换行
> i = 3:  j = 1, 2, 3    → 3 个式子,换行
> ...
> i = 9:  j = 1, 2, ..., 9 → 9 个式子,换行
> ```

**常见错误**

> 1. **错误现象**:忘记外层循环结束后 print() 换行
>    **原因:**内层用 end="" 不换行,外层结束必须 print() 换行
> 2. **错误现象**:嵌套循环变量名重复
>    **原因:**内外层变量要不同(通常用 i 和 j)。修正:外层 i,内层 j

#### 综合实战 —— 鸡兔同笼 & 菜单循环

> **痛点**:学了循环,不知道怎么用在实际问题里。
> **类比**:鸡兔同笼就像"穷举法" —— 把所有可能都试一遍,找到正确答案。
> **解释**:循环 + 条件 = 解决实际问题的利器。

```python
# 鸡兔同笼:笼头 35 个,笼脚 94 只,鸡兔各几只?
# 鸡 2 脚,兔 4 脚
for chicken in range(36):          # 鸡 0 到 35
    rabbit = 35 - chicken          # 兔 = 总数 - 鸡
    if chicken * 2 + rabbit * 4 == 94:
        print(f"鸡 {chicken} 只,兔 {rabbit} 只")
        break

# 菜单循环:while True + break
while True:
    print("1.查询 2.存款 3.取款 0.退出")
    choice = input("请选择:")
    if choice == "0":
        print("再见!")
        break
    elif choice == "1":
        print("查询余额")
    elif choice == "2":
        print("存款")
    elif choice == "3":
        print("取款")
    else:
        print("无效选择")
```

**逐行解剖**

- `for chicken in range(36)` = 穷举鸡的数量(0 到 35)
- `if chicken * 2 + rabbit * 4 == 94` = 判断脚数是否匹配
- `while True` = 无限循环,靠 break 退出
- `break` = 用户选 0 时退出
