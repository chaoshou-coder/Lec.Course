# 04 · 试题集组织方法 —— 如何为每天的学习设计三层试题集

> **归属阶段:** BUILD (状态机 `BUILD` 态参考,每个 build subagent 使用)
> **原始文档:** Lec.Python `dev/skills/04_试题集组织方法.md`
> **用途:** 为每个知识点系统设计当堂练(practice) + 课后作业(task) + 综合项目(weekly project)三层试题集。
> **适用场景:** build subagent 制课过程中,为每天的 lesson 产出配套练习。

---

## 1. 题型设计:★~★★★★★ 五级难度模型

| 难度 | 名称 | 典型描述 | 知识点数 | 学员动作 |
|---|---|---|---|---|
| ★ | 纯语法复制 | "照着示例写一遍" | 1 | 记忆 |
| ★★ | 单步变形 | "把示例的数值改一下" | 1 | 理解 |
| ★★★ | 组合 | "把两个知识点组合起来" | 2-3 | 应用 |
| ★★★★ | 抽象建模 | "把实际问题抽象成代码" | 3-4 | 分析 |
| ★★★★★ | 综合项目 | "做一个完整的小项目" | 5+ | 创造 |

### 与 DAG 节点最低要求的映射

每个 DAG 节点有 `min_requirement` 标注,决定了 BUILD 态应产出的练习深度:

| min_requirement | 节点教学深度 | 对应习题天花板 |
|---|---|---|
| 了解 | 能说出是什么、解决什么问题 | ★★ 题(单步变形) |
| 能写 | 能独立写出代码实现 | ★★★ 题(组合应用) |
| 能教 | 能给别人讲清楚原理 | ★★★★ 题(抽象建模) |

build subagent 在为某个节点设计习题时,难度不应超过该节点的 min_requirement 对应的"天花板"。

---

## 2. 三层题型结构

| 题型层级 | 数量 | 难度分布 | 强制性 |
|---|---|---|---|
| 当堂练(in_class/practice) | 6 道/天 | ★×2 + ★★×2 + ★★★×2 | 必做 |
| 课后作业(homework/task) | 3 道/天 | ★★×1 + ★★★×1 + ★★★★×1 | 选做 |
| 综合项目(weekly project) | 1 道/周 | ★★★★★ | 必做 |

**设计逻辑:**
- 当堂练以 ★~★★★ 为主,保证 80% 学员能在课上完成
- 课后作业以 ★★★~★★★★ 为主,需要思考,标注为"选做"
- 综合项目是 ★★★★★,需要综合多天知识,有验收标准

## 3. 当堂练(6 道/天)的设计要点

### 3.1 覆盖边界

每道题必须覆盖以下边界中的至少一个:

| 边界类型 | 示例 | 常见错误 |
|---|---|---|
| 空输入 | `[]` / `""` / `{}` | 没处理空,直接崩溃 |
| 零 | `0` / `0.0` | 除以零错误 |
| 负数 | `-1` / `-100` | 逻辑不考虑负数 |
| 单元素 | `[1]` / `"a"` | 只考虑多元素情况 |
| 重复元素 | `[1, 1, 1]` | 没考虑重复 |
| 极大/极小值 | `999999` / `-999999` | 溢出或逻辑错误 |
| 类型混合 | `[1, "a", None]` | 假设输入类型一致 |

### 3.2 线性递进

当堂练的 6 道题必须"线性递进":
```
题 1:纯语法复制(★)   → 建立信心
题 2:简单变形(★)     → 巩固语法
题 3:加一个边界(★★)  → 开始思考
题 4:组合两个知识点(★★) → 知识整合
题 5:小综合(★★★)   → 独立思考
题 6:综合应用(★★★)  → 挑战题(可做可不做)
```

时间预算:题 1-2 在 5 分钟内,题 3-4 在 10 分钟内,题 5-6 在 15 分钟内。

### 3.3 当堂练文件模板

每个 practice 文件必须包含以下结构(这是 QA 态 structural_check 的校验对象):

```python
"""
[难度: ★★]
[所属知识点: 列表 CRUD]
[预计完成时间: 8 分钟]

题目描述(中文,场景化,可独立运行)

示例:
    >>> 调用方式 / 示例输入 → 示例输出
"""

# ======================
# 学员代码区(以 pass 作为占位符)
# ======================
pass

# ======================
# 测试区(可复制到终端验证)
# ======================
if __name__ == '__main__':
    # 测试 1:正常输入
    # 测试 2:空输入
    # 测试 3:边界
    pass
```

约束:
- 所有字符串中文,≤ 100 字符/行
- 学员代码区用 `pass` 占位(或部分脚手架),**不用学员没学过的语法**
- 测试区 ≥ 2 个用例,覆盖边界

## 4. 课后作业(3 道/天)的设计要点

### 4.1 综合运用

每道课后作业必须综合运用**当天 + 前一天**的知识。

```
Day 5 课后作业:
  题 1(Day 5 知识):字符串 replace 应用 → ★★★
  题 2(Day 5 + Day 4):字符串 + 循环 → ★★★
  题 3(Day 5 + Day 4 + Day 3):字符串 + 循环 + 分支 → ★★★★
```

### 4.2 可做可不做(选做)

课后作业标注为"选做",不强制完成。让有能力的学员挑战,不打击能力弱的学员。

---

## 5. 综合项目(1 道/周)的设计要点

综合项目是 ★★★★★,需要:
- 综合 3-5 天的知识
- 有明确的验收标准(checklist)
- 有 README.md 描述项目背景和功能
- 有 step-by-step 的完成指引
