# 05 · 自学材料排版规范 —— build subagent 产出笔记时必须遵守的格式标准

> **归属阶段:** BUILD (每个 build subagent 强制参考) + QA (structural_check 依据)
> **用途:** 所有自学笔记材料(self-study notes,取代课堂 slides)必须遵守的排版标准。这是 QA 态 structural_check 的直接依据。
> **版本:** v3.0(2026-07-14),以 OOP 封装/继承/多态/组合 notebook 为 gold standard 重写
> **关键升级:** 8 步趁热打铁循环 + ASCII 内存图 + NCDL Break It + 常见错误

---

## 1. 标题层级:h3(天标题) + h4(知识点小节)

```
### Day 08 · OOP 基础                         ← h3(一天内唯一)
#### __init__ 构造函数 —— 给对象"贴标签"       ← h4(小节)
#### 实例方法 —— 对象能做什么                  ← h4(小节)
**逐行解剖**                                    ← 正文加粗(不是标题)
**为什么:**这是解释,不是标题                    ← 正文加粗
```

**规则:**
- `###` 只用一次(当天标题)
- `####` 用于每个知识点的标题
- 不再往下嵌套(不用 `#####`)
- 解释性文字用 **加粗** + 正文,不额外加标题层级

---

## 2. 加粗:只加关键词

```
✅ **类** = 蓝图/模板
✅ 使用 **append()** 方法可以在末尾添加元素
❌ **类是蓝图模板,用来创建对象**(整句加粗)
```

**规则:**
- 每行最多 1 处加粗
- 加粗对象:关键词、方法名、重要概念
- 不整句加粗,除非是"核心定义"

## 3. 一个概念 = 一个代码块(or fenced code in notes)

每个示例代码独立一个块,学员可以逐个理解:
```
✅ print("Hello")      ← 块 1:基础用法
✅ print("你好")       ← 块 2:中文
✅ print("A", "B")    ← 块 3:多值

❌ 以上 4 行全放在一个块里(输出混在一起,看不出每个独立结果)
```

### 3.1 代码注释密度(v1.2 R12)

**规则:** 示例代码的每一行(或每个逻辑行)必须有中文注释。

```python
# ✅ 正确:每行有注释
class Student:                    # 定义类
    def __init__(self, name):     # 构造函数,self 是新对象
        self.name = name          # 把参数 name 绑到对象

# ❌ 错误:只有部分行有注释
class Student:
    def __init__(self, name):     # 构造函数
        self.name = name          # 把参数 name 绑到对象
```

**为什么:** 初学者需要每行都有解释,否则会卡在某一行的理解上。

## 4. 每个知识点 = 完整的 8 步趁热打铁循环

每个完整知识点必须包含以下 8 步,**缺一不可**(QA 态 structural_check 逐项验证):

```
① 概念 md:痛点(为什么需要) → 类比(生活例子) → 解释(是什么)
② ASCII 内存图 md(可选但推荐):文字画内存关系(见 4.2)
③ 代码示例 code:每行中文注释 + 执行过程跟踪(必须内嵌在代码块中,见 4.1)
④ 逐行解剖 md:逐行解释语法和参数
⑤ 常见错误 md:学员最容易犯的错(≥2 条,含具体报错信息)
⑥ 苏格拉底引导 md:问自己(≥3 个引导问题)
⑦ 学员代码区 code:pass 占位
⑧ 参考答案 code:完整可运行代码 + 注释
```

### 4.1 执行过程跟踪格式(最关键)

执行过程跟踪必须内嵌在代码块中,使用**编号步骤**格式(①②③④):

```python
password = ""
while password != "123":
    password = input("请输入密码:")
print("密码正确,欢迎!")

# --- 执行过程 ---
# 第 1 行 password = "":
#   ① 创建 password,初值空字符串
#
# 第 2 行 while password != "123":
#   ① "" != "123" → True → 进循环
#
# 第 3 行 password = input(...):
#   ① 显示提示,等待用户输入
#   ② 把输入赋给 password
#
# 回到第 2 行再判断:
#   ① 若输入 "abc" → True → 继续循环
#   ② 若输入 "123" → False → 退出循环
```

**规则:**
- 每行代码必须有编号步骤(①②③④),不能只写"第X行:做什么"
- 步骤要详细到"内存/流程变化"层面(如"分配新内存""修改变量指向")
- 循环/条件必须展示"回到某行再判断"的过程

**执行跟踪 ≠ 逐行解剖。** 执行跟踪(代码块内注释):逐步解释执行时内存/流程变化。逐行解剖(md):解释语法和参数含义。两步不能合并。

---

### 4.2 ASCII 内存图规范(v1.2)

ASCII 图用于直观展示内存关系、执行流程或数据转换。

**规则:**
1. **只用 plain ASCII 字符** —— 不用 Unicode box-drawing(┌┐└┘│─├┤├┬┴┼),不用制表符,只用 `- | + = * # / \ < > ( ) [ ]` 和空格
2. **宽度 ≤ 60 字符** —— 确保在手机/窄屏上也能完整显示
3. **高度 ≤ 15 行** —— 过长的图失去"一眼看懂"的价值
4. **必须有标题和关键说明** —— 图上方有标题(如 `**ASCII 内存图 —— __init__ 执行过程**`),图下方有"关键"总结

**推荐风格(简单方框+箭头):**

```
__init__ 执行过程:

  stu1 = Student("张三", 18)
       |
       v
  +---------------------+
  | 1. 分配新内存        |
  | 2. 调用 __init__     |
  | 3. self.name = "张三"|
  | 4. self.age = 18     |
  +---------------------+
       |
       v
  stu1 ---> name="张三", age=18
```

**不推荐(复杂 box-drawing):**

```
  +----------------------------------------------------+
  |  __init__(self, name, age)                          |
  |    self.name = name   <-- 把参数绑到对象             |
  |    self.age = age                                   |
  +----------------------------------------------------+
```

**为什么不用 Unicode box-drawing:** 不同渲染器(VS Code/GitHub/终端)对宽字符宽度处理不一致,会导致对齐混乱。Plain ASCII 在所有环境下表现一致。

## 5. 常见错误格式(v1.2 R12)

```markdown
> **常见错误**
> 1. **错误现象**:`TypeError: unsupported operand type(s) for +: 'int' and 'str'`
>    **原因**:`age` 是字符串,不能直接和整数相加。修正:`print(int(age) + 5)`
> 2. **错误现象**:`AttributeError: 'Product' object has no attribute 'title'`
>    **原因**:`__init__` 里漏写 `self.title = title`。修正:补上
```

**规则:**
- 每条必须包含**具体的 Python 报错信息**(如 `TypeError: ...`),不能写"会报错"
- 必须说明**原因**和**修正方式**
- 每个知识点 ≥ 2 条常见错误

## 6. NCDL Break It 模式(负案例驱动学习,v1.2 R12 强制)

**强制规则:** 当 `learning-plan.md` 中节点标注 `teaching_method: ncdl` 时,
该知识点的 notes.md **必须包含** Break It 演示段。

格式:

```python
# ============ BREAK IT 演示 ============
class BrokenAlipay:
    pass  # 漏写 execute 方法!

try:
    checkout(99.0, BrokenAlipay())
except AttributeError as e:
    print(f"报错: {e}")
# ============ END BREAK It ============
```

**规则:**
- Break It = 故意写错代码 → 展示 Traceback → 引出正确做法
- 每个 NCDL 节点 ≥ 1 段 Break It 演示
- 适用于:所有有反模式的模块(OOP/循环/函数/异常)
- 不适用于:纯数学/概念(如反向传播推导)

---

## 6.1 每个知识点必须有学员代码区(v1.2 R12 新增)

**规则:** 每个知识点(不仅是每天末尾)必须有独立的"学员代码区 + 参考答案"对。

格式(嵌入在知识点末尾):

```markdown
# ============ 学员代码区 ============
# 请定义一个 Product 类(暂时用 pass),
# 创建两个对象 p1、p2,
# 分别给它们绑定 title 和 price,
# 然后打印两个商品的 title。
class Product:
    pass

# p1 = Product()
# p1.title = ...
pass

# ============ 参考答案 ============
class Product:
    pass

p1 = Product()
p1.title = "Python 入门"
p1.price = 59.8

print(p1.title)  # Python 入门
```

**为什么:** 学员学完一个概念后,立即练习才能巩固。
只在最后留练习 = 前面概念没有即时巩固。

## 7. 知识点顺序约束(无日程规划)

框架不强制每日学习顺序,但**知识点有前置依赖**。编写某个知识点的材料前,必须确认其前置知识点已经覆盖,不能使用学员还没学的语法。

**示例(Python 核心知识点序列):**

| 知识点序号 | 主题 | 已教语法 | 禁止使用的语法 |
|---|---|---|---|
| 01 | 入门 | print/input/变量/类型 | def/class/循环/列表/字典 |
| 02 | 字符串 | +字符串/索引/切片/f-string | def/class/循环/列表/字典 |
| 03 | 条件分支 | +if/elif/else | def/class/循环/列表/字典 |
| 04 | 循环 | +while/for/range | def/class/列表/字典 |
| 05 | 函数 | +def/函数 | class/列表/字典 |
| 06 | 容器 | +列表/字典 | class |
| 07+ | OOP | +class/OOP | 外部库 |

**关键:** 编写任何材料前必须对照 `learning-plan.md` 确认该知识点的**前置依赖已覆盖**,不能假设学员已经学会后续知识点。学员按自己的节奏学习,但知识点的依赖关系必须被尊重。

## 8. 知识点末尾必须有"下一知识点衔接"段(v1.2)

每个知识点 MD 末尾必须有导航链接(≤3 行):

```
---

[← 上一个知识点](../knowledge/prev-topic.md) | [返回知识地图](../README.md) | [下一个知识点 →](../knowledge/next-topic.md)
```

**规则:**
- 第一个知识点没有"上一个",链接指向 README
- 最后一个知识点没有"下一个",链接指向 README
- 链接使用相对路径 `../knowledge/xxx.md`

**为什么:** 学员按自己的节奏学习,需要清晰的导航在不同知识点间跳转。

---

## 10. 知识地图链接规范(v1.2)

课程入口 README.md 中的知识地图必须与具体知识点双向链接。

### 10.1 README 知识地图 → 知识点链接

README 中的知识地图表格,每个知识点名称必须链接到对应的 MD 文件:

```markdown
## 知识点地图

| # | 知识点 | 文件 |
|---|--------|------|
| 01 | [Python 入门](knowledge/01-python-入门.md) | `knowledge/01-python-入门.md` |
| 02 | [字符串与格式化](knowledge/02-字符串与格式化.md) | `knowledge/02-字符串与格式化.md` |
```

### 10.2 知识点 MD 内的知识地图 → 章节锚点

每个知识点 MD 顶部的知识地图表格,每个知识点名称必须链接到该 MD 内的章节锚点:

```markdown
**本课知识地图**

| 知识点 | 解决什么问题 |
|--------|-------------|
| [print() 输出](#print-----让电脑把内容显示到屏幕上) | 让电脑把内容显示到屏幕上 |
```

### 10.3 知识点末尾 → 返回 README 链接

每个知识点 MD 末尾必须有返回主导航的链接:

```markdown
---

[← 上一个](../knowledge/prev.md) | [返回目录](../README.md) | [下一个 →](../knowledge/next.md)
```

**为什么:** 学员需要清晰的导航:从 README 进入任意知识点,从知识点返回 README,在知识点间按自己节奏跳转。
